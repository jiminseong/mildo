# 제안용 독립 데모 앱 작업 계획

기준 문서: `OVERVIEW.md`

목표:
- 공고 이해를 말이 아니라 화면과 상호작용으로 바로 보여줄 수 있는 제안용 데모를 만든다.
- 무회원 검사 진입, 순차 문항 응답, 즉시 결과 리포트, PDF 저장 흐름, 관리자 로그 확인까지 핵심만 압축해 보여준다.
- 기존 엑셀 계산 로직을 웹 서비스 분석 엔진으로 옮겨야 하는 프로젝트라는 점을 UI와 설명 카드에서 함께 드러낸다.

구현 위치:
- 앱 루트: `wishket/심리검사서비스프로토타입개발/proposal-demo`
- 화면 코드: `proposal-demo/src/app/*`
- 데모는 메인 서비스 앱과 분리된 독립 Next.js 앱으로 유지한다.

생성 기준:
- `npx create-next-app@latest proposal-demo --ts --eslint --tailwind --app --src-dir --import-alias "@/*" --use-npm --yes`
- 한글 경로 이슈를 피하기 위해 `dev`/`build` 스크립트는 `--webpack` 기준으로 둔다.
- 배포는 별도 Vercel 프로젝트로 관리하고, 댓글/메시지에는 `제안용 예시 데모`라고 명확히 적는다.

데모 범위:
- 회원가입 없이 바로 검사에 들어가는 공개 진입 경험
- 1문항씩 진행하는 순차 응답 UI
- 응답에 따라 결과 지표와 해석이 바뀌는 실시간 리포트
- 브라우저 인쇄 기반 PDF 저장 예시
- 엑셀 로직 이관 포인트를 보여주는 분석 엔진 설명 카드
- 관리자용 접속 로그 확인 테이블과 연구용 통계 처리 예시
- 데스크탑 우선, 모바일에서도 무너지지 않는 반응형 레이아웃

비범위:
- 실제 DB 저장
- 실제 엑셀 파일 업로드/파싱
- 실제 통계 엔진 연동
- 최종 납품용 PDF 템플릿 확정
- 실제 관리자 인증/권한 제어
- 상용화 단계의 회원 시스템, 운영 CMS, 보안 고도화

배포 메모:
- 로컬 실행: `npm run dev`
- 사전 검증: `npm run lint`, `npm run build`
- Vercel에서는 `wishket/심리검사서비스프로토타입개발/proposal-demo` 디렉토리를 루트로 연결한다.
- 공개 설명에는 실제 서비스가 아니라 `연구과제 방향 제안을 위한 프로토타입 데모`라는 점을 분명히 남긴다.

디자인 방향:
- 의료 서비스처럼 차갑거나, 일반 SaaS처럼 번쩍이는 느낌을 피하고 연구·상담 보조 도구처럼 차분하게 구성
- 오프화이트 배경, 얇은 선, 세리프 헤드라인, 정보 밀도 조절로 사람이 다듬은 편집형 화면을 만든다
- 과한 그라데이션, 유광 카드, 의미 없는 추상 아이콘을 피하고 문항/리포트/로그 자체가 중심이 되게 한다
- `AI가 만든 무난한 랜딩`보다 실제 제품 콘셉트 시안처럼 보이도록 흐름과 정보 구조를 우선한다
