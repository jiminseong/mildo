# 심리검사 서비스 프로토타입 개발 지원서 초안

## 기본 입력값

- 지원 금액: 5,000,000원
- 지원 기간: 30일
- 관련 포트폴리오: 있습니다

## 제출용 본문

```text
<프로젝트 진행 제안>
[프로젝트 이해]
- 본 프로젝트는 연구과제 수행을 위한 웹 기반 심리검사 프로토타입 구축으로 이해했습니다.
- 공고 기준 핵심은 회원가입 없이 누구나 검사에 참여할 수 있는 PC 웹 검사 흐름, 응답 데이터 저장, 기존 엑셀 계산 로직 자동화, 즉시 결과 리포트/PDF 제공, 관리자 로그 확인 화면까지를 하나의 서비스로 정리하는 것입니다.
- 특히 이번 프로젝트는 일반적인 설문 페이지 제작보다, 현재 엑셀에 들어 있는 계산 공식과 조건식, 통계 처리 로직을 웹 환경에서 동일하게 재현하는 작업이 가장 중요해 보입니다.
- 즉 화면 구현보다 먼저 문항 구조, 역채점 여부, 척도별 합산 방식, 결과 해석 문구, t-test 처리 기준을 정확히 문서화하는 단계가 필요합니다.

[구축 범위 제안]
- 현재 공고 범위 기준 1차 구축은 검사 진입 화면, 순차 문항 응답 UI, 결과 리포트 화면, PDF 저장, 응답 데이터 저장, 관리자 로그/조회 화면, 기본 서버·DB·배포 구성까지로 보는 것이 적절합니다.
- 연구용 프로토타입 단계에서는 회원 시스템, 복잡한 운영 CMS, 상용 결제/마케팅 기능보다 검사 정확도와 결과 재현성을 우선해야 합니다.
- 제공 예정인 문항 목록과 엑셀 원본이 정리되어 있다면 30일 안에 1차 프로토타입 범위를 충분히 맞출 수 있다고 판단합니다.

[기술 제안]
- 프런트엔드는 Next.js + TypeScript + Tailwind CSS 기반으로 구성해 검사 흐름, 결과 리포트, 관리자 화면까지 하나의 코드베이스로 정리하는 방식을 우선 제안드립니다.
- 데이터 저장과 로그 관리는 PostgreSQL 기반으로 설계하고, 초기 프로토타입에서는 Supabase 또는 일반 Node 백엔드 구조 중 운영 편의가 높은 방식을 선택할 수 있습니다.
- 엑셀 계산 공식과 통계 처리 범위가 단순 합산/조건 분기 수준인지, 별도 통계 라이브러리 수준까지 필요한지에 따라 분석 로직은 TypeScript 단일 구성 또는 Python 분석 모듈 분리 여부를 판단하는 것이 좋습니다.
- 시작 전에 엑셀 시트 구조를 먼저 검토한 뒤, 화면보다 계산 로직 검증용 테스트 케이스를 먼저 만드는 방식으로 정확도를 확보하겠습니다.

[예상 일정]
- 1주차: 문항/결과 구조 정리, 엑셀 로직 분석, 데이터 모델 및 정책 정의
- 2주차: 화면 설계, 검사 플로우 구현, 관리자 로그 기본 구조 구현
- 3주차: 결과 분석 로직, 리포트 화면, PDF 저장 기능 구현
- 4주차: QA, 계산 결과 검증, 수정 반영, 배포 및 산출물 정리

[착수 전 확인 사항]
- 엑셀 파일 내 공식, 숨김 시트, 매크로 사용 여부
- 척도별 결과 해석 기준과 리포트 문구 존재 여부
- t-test 적용 대상 데이터와 계산 방식
- PDF 결과물에 필요한 고정 레이아웃 유무
- 관리자 페이지에서 확인해야 할 로그 항목 범위

[공유 및 산출물]
- 진행 중에는 단계별 데모와 문서로 계산 로직 검증 결과를 먼저 공유해, 화면보다 중요한 결과 정확도를 빠르게 맞추겠습니다.
- 산출물은 화면 설계안, 기능 명세, 서버/DB 구조 정리, 전체 소스 코드 기준으로 정리 가능합니다.
- 필요하시면 미팅 전에도 핵심 흐름과 결과 리포트 방향을 예시 화면으로 먼저 공유드릴 수 있습니다.

<유사 프로젝트 진행 경험>
[AIM 앱 패밀리 설계·운영 경험]
- 직접적인 심리검사 상용 서비스를 과장해서 말씀드리진 않겠습니다.
- 다만 AIM 앱 패밀리를 직접 기획·설계·개발·운영하면서 사용자 데이터 구조 설계, 기록 기반 결과 화면, Supabase/PostgreSQL 기반 저장 구조, 서비스별 흐름 분리를 직접 수행했습니다.
- 이번 프로젝트에서도 문항 응답 데이터, 결과 지표, 리포트 출력 구조를 안정적으로 설계하는 데 바로 적용 가능합니다.

[포토XX 키오스크 + 관리자 시스템 경험]
- 포토XX 프로젝트에서는 불특정 다수가 바로 사용하는 현장형 사용자 흐름과 운영자용 관리자 시스템을 함께 구축했습니다.
- 사용자 입장에서 복잡한 진입 절차 없이 빠르게 사용해야 하는 화면과, 운영자가 로그와 상태를 확인하는 화면을 분리 설계한 경험이 있어 이번 프로젝트의 검사 화면과 관리자 로그 화면 구성에도 실무적으로 적용 가능합니다.

[데이터 기반 프로토타입 접근 경험]
- 기능 수가 많지 않더라도, 실제로는 데이터 구조와 결과 계산 방식이 핵심인 유형의 프로젝트들을 우선 구조화하고 구현해온 편입니다.
- 이번 공고도 단순 퍼블리싱보다 계산식 재현과 결과 신뢰도가 더 중요한 프로젝트로 보고 접근하겠습니다.

<사용 기술과 툴>
[기술 스택]
- 프런트엔드: Next.js, React, TypeScript, Tailwind CSS
- 백엔드/DB: PostgreSQL(Supabase 포함 가능), API Route 또는 별도 서버 구조
- 분석/리포트: Excel 로직 이관, 통계 계산 모듈, PDF 생성

[진행 방식]
- 기획과 개발을 동시에 밀기보다, 먼저 문항 구조와 계산 로직을 검증 가능한 형태로 정리한 뒤 화면과 결과 리포트를 구현합니다.

[커뮤니케이션]
- 위시켓 메시지와 정기 미팅 기준으로 진행하고, 계산 검증이 필요한 부분은 샘플 데이터와 비교표로 빠르게 확인드리겠습니다.

[산출물]
- 화면 설계서, 기능 명세, 서버/DB 설계 내역, 개발 소스 코드, 배포 가이드 기준으로 전달 가능합니다.

관련 포트폴리오:
- AIM 앱 패밀리: 사용자 데이터 구조, 결과 화면, Supabase/PostgreSQL 기반 서비스 설계 및 운영 경험
- 포토XX 키오스크: 불특정 사용자 진입 흐름과 운영 관리자 화면을 함께 구축한 경험
- 제천남여헬스클럽/새영동숯불갈비: 실제 운영 가능한 반응형 웹사이트를 기획·디자인·개발까지 일괄 수행한 경험
```

## 작성 메모

- 지원 금액과 기간은 공고 제시 범위에 맞췄습니다.
- 이 프로젝트는 화면보다 계산식 이관 정확도가 핵심이므로, 지원서에서도 그 포인트를 전면에 두었습니다.
- 엑셀 원본 구조가 복잡할 수 있으므로, 착수 전 검토 항목을 분명히 적어 과소산정처럼 보이지 않게 정리했습니다.
- 심리검사 직접 납품 경험은 과장하지 않고, 데이터 구조·결과 화면·관리자 시스템 경험으로 연결했습니다.
