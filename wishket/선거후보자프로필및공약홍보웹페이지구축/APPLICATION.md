# 선거 후보자 프로필 및 공약 홍보 웹페이지 지원서 초안

## 기본 입력값

- 지원 금액: 500,000원
- 지원 기간: 3일
- 관련 포트폴리오: 있습니다

## 제출용 본문

```text
<프로젝트 진행 제안>
[프로젝트 이해]
- 본 프로젝트는 선거 기간 동안 후보자의 프로필, 핵심 공약, 활동 사진을 빠르게 전달하는 반응형 홍보 웹페이지 구축으로 이해했습니다.
- 공고 기준 핵심은 기능이 많은 서비스가 아니라, 한정된 기간 안에 후보자의 이미지와 메시지를 명확하게 보여주는 정보 전달 구조를 만드는 것이라고 봤습니다.
- 특히 모바일에서 읽기 쉬운 구성, 후보자 소개와 공약 우선순위 정리, 활동 사진 신뢰감 확보가 중요하며, 단일 스크롤 페이지 또는 2~3개 페이지 구성 중 어떤 방식이 더 적합한지도 초기에 바로 정리해야 합니다.
- 선거 일정상 개발 기간이 짧기 때문에, 범위를 넓히기보다 메시지가 가장 선명하게 보이는 1차 MVP 구조로 빠르게 완성하는 접근이 적절해 보입니다.

[구축 범위 제안]
- 현재 예산과 일정 기준으로는 단일 스크롤 반응형 웹페이지를 1차 기본안으로 제안드립니다.
- 1차 범위는 메인 히어로 영역, 후보자 프로필, 핵심 공약, 활동 사진/갤러리, 기본 CTA, PC/Mobile 반응형 대응까지로 보고 있습니다.
- 공약 상세를 너무 길게 풀기보다, 핵심 공약 요약과 상세 설명의 균형을 맞춰 한 화면 안에서 빠르게 읽히도록 구성하겠습니다.
- 게시판, CMS, 뉴스/보도자료 관리, 다국어, 관리자 기능은 현재 공고 범위와 일정 기준에서는 제외하고 필요 시 2차 범위로 분리하는 편이 적절합니다.

[기술 제안]
- 짧은 일정과 안정적인 배포를 고려해 Next.js + TypeScript + Tailwind CSS 기반으로 제안드립니다.
- 데이터 입력이 복잡한 서비스가 아니라 콘텐츠 중심 페이지이므로, 초기에는 정적 구조로 빠르게 제작하고 선거 기간 중 문구/사진 교체가 쉽도록 구성하는 것이 효율적입니다.
- 기본적인 메타 정보, 공유용 대표 문구, 반응형 레이아웃, 이미지 최적화까지 함께 정리해 실제 홍보용 페이지로 바로 활용할 수 있게 구현하겠습니다.

[예상 일정]
- 1일차: 요구사항 정리, 페이지 구조 확정, 시안 방향 제안
- 2일차: 디자인 반영, 프로필/공약/갤러리 퍼블리싱
- 3일차: 반응형 보완, 수정 반영, 최종 배포 준비

[착수 전 확인 사항]
- 단일 스크롤 1페이지로 갈지, 2~3개 페이지로 나눌지
- 후보자 공식 사진 원본 해상도
- 최우선 노출 공약 3~5개
- 활동 사진 수량과 캡션 여부
- 도메인/호스팅 연결 여부 및 공개 시점

[공유 및 산출물]
- 진행 중에는 1차 시안과 데모 링크를 빠르게 공유해 방향 확인 시간을 줄이겠습니다.
- 산출물은 화면 구성안, 반응형 웹 구현본, 개발 소스 기준으로 정리 가능합니다.
- 디자인 원본은 협의한 방식 기준으로 정리하고, 개발 이후에도 문구/사진 교체가 어렵지 않도록 구조를 깔끔하게 맞추겠습니다.

<유사 프로젝트 진행 경험>
[정보 전달형 반응형 웹 구축 경험]
- 제천남여헬스클럽 프로젝트에서는 브랜드 소개, 운영 철학, 시설 안내, 위치/문의 흐름을 한 화면 안에서 이해하기 쉽게 정리하는 반응형 랜딩 페이지를 기획·디자인·개발까지 일괄 진행했습니다.
- 새영동숯불갈비 프로젝트에서는 매장형 브랜드 사이트 구조를 설계하며, 복잡한 기능보다 사진, 메시지, 동선이 먼저 읽히는 정보 구조를 중심으로 구축했습니다.
- 두 프로젝트 모두 공통적으로 `짧은 시간 안에 핵심 메시지를 잘 보이게 만드는 페이지 설계`가 중요했고, 이번 공고에도 같은 접근을 바로 적용할 수 있습니다.

[실무 적용 포인트]
- 이번 프로젝트는 후보자 신뢰감과 공약 전달력이 핵심이므로, 일반적인 템플릿 랜딩처럼 보이지 않도록 톤을 잡는 것이 중요하다고 보고 있습니다.
- 후보자 사진, 이름, 핵심 문장, 대표 공약이 첫 화면에서 분명히 읽히도록 하고, 활동 사진은 단순 갤러리가 아니라 후보자의 현장성을 보강하는 흐름으로 배치하겠습니다.
- 일정이 짧은 만큼 초기 기획에서 섹션 우선순위를 정확히 정하면 완성도와 속도를 함께 확보할 수 있습니다.

<사용 기술과 툴>
[기술 스택]
- 프론트엔드: Next.js, React, TypeScript, Tailwind CSS
- 배포: Vercel 또는 협의된 정적 호스팅 환경

[진행 방식]
- 요구사항과 자료를 받은 뒤 페이지 구조와 디자인 방향을 먼저 빠르게 확정하고, 이후 반응형 퍼블리싱과 수정 반영을 짧은 주기로 진행합니다.

[산출물]
- 화면 구성안, 반응형 웹 구현본, 개발 소스 코드, 배포 가이드 기준으로 정리 가능합니다.
```

- 예산과 일정이 매우 짧기 때문에, `빠르고 완성도 있는 1페이지 홍보 웹`을 1차 제안 범위로 명확히 잡았습니다.
- 과장해서 기능 범위를 넓히기보다, 정보 구조·디자인 톤·모바일 가독성을 강점으로 보이게 정리했습니다.
- 선거 홍보물 성격상 첫 화면 임팩트와 공약 정리 방식이 중요하므로, AI 템플릿처럼 보이지 않는 편집형 디자인 방향을 함께 강조했습니다.
