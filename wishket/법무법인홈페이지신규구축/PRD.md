# 법무법인 한결 홈페이지 PRD (최종본)

## 1. 문서 개요

- 프로젝트명: 법무법인 한결 홈페이지/관리자 데모 구축
- 대상 폴더: wishket/법무법인홈페이지신규구축
- 구현 앱: wishket/법무법인홈페이지신규구축/proposal-demo
- 문서 목적: 합의된 상세 기능 요구사항(2-1~2-5)의 충족 여부를 기준으로 현재 상태를 점검하고, 누락 기능의 구현 기준을 명확히 정의한다.

## 2. 상세 기능 요구사항 점검 결과

### 2-1. 법무법인 소개 페이지

요구사항:
- 법무법인 소개
- 변호사 소개
- 연혁
- 오시는 길

현재 상태: 부분 충족
- 충족: 법무법인 소개, 변호사 소개, 오시는 길
- 미충족: 연혁 섹션 부재

근거 화면:
- 소개 페이지: /about

보완 필요:
- 연혁 타임라인(연도/이벤트) 섹션 추가

### 2-2. 전문 분야 상세 페이지

요구사항:
- 성범죄, 마약 주제별 상세 페이지
- 관련 법률 정보
- 사건 유형
- 대응 절차
- 판례 등 전문 콘텐츠

현재 상태: 부분 충족
- 충족: 성범죄/마약 상세 분리, 사건 위험 포인트, 대응 절차, FAQ
- 미흡: 명시적 판례 섹션 부재, 관련 성공사례 연결 약함

근거 화면:
- 목록: /practice
- 상세: /practice/sexual-crime, /practice/drug-crime

보완 필요:
- 판례/참고 사례 섹션 추가
- 관련 성공사례 링크 블록 추가

### 2-3. 성공사례 관리

요구사항:
- 성공사례를 체계적으로 노출
- 관리자가 성공사례 등록/수정/삭제 가능

현재 상태: 부분 충족
- 충족: 성공사례 목록/상세 노출 구조 구축
- 미충족: 성공사례 데이터 자체의 관리자 CRUD 미연동

근거 화면:
- 목록/상세: /cases, /cases/[slug]
- CMS 데모: /cms (현재는 일반 콘텐츠 CRUD)

보완 필요:
- CMS에서 성공사례 엔터티(Case) CRUD 지원
- /cases 화면이 CMS 저장 데이터(localStorage) 기준으로 동작하도록 연결

### 2-4. 온라인 상담 신청

요구사항:
- 온라인 상담 신청 폼

현재 상태: 충족
- 사건 분야, 긴급도, 사건 개요, 동의 항목, 제출 후 안내 메시지 구현 완료

근거 화면:
- 상담 신청: /consultation

### 2-5. 콘텐츠 관리 시스템(CMS)

요구사항:
- 관리자 콘텐츠 관리 기능

현재 상태: 충족 (데모 범위)
- 로컬스토리지 기반 콘텐츠 등록/수정/삭제/리셋 구현 완료
- 데모 목적의 클라이언트 단 저장소 사용

근거 화면:
- CMS 데모: /cms

제약:
- 인증/권한/서버 저장 미구현

## 3. 최종 범위 정의

### 3.1 데모 범위 (이번 단계)

- 메인 랜딩 및 공통 레이아웃(헤더/푸터/네비게이션)
- 법무법인 소개
- 전문분야 목록/상세
- 성공사례 목록/상세
- 온라인 상담 신청 폼
- 로컬스토리지 기반 CMS 데모

### 3.2 보완 범위 (요구사항 완전 충족을 위한 추가)

- 소개 페이지 연혁 섹션
- 전문분야 상세의 판례/관련 성공사례 섹션
- 성공사례 전용 CMS CRUD 및 /cases 연동

### 3.3 제외 범위 (현 단계)

- 서버 DB 영구 저장
- 관리자 인증/권한
- 실운영 알림 연동(메일/문자/카카오)
- 실데이터 마이그레이션

## 5. 기능 요구사항 (최종)

### FR-01 공통 레이아웃
- 전 라우트에서 헤더/푸터 일관 유지
- 헤더에서 주요 페이지와 CMS 데모 바로가기 제공
- 홈(/)에서는 뒤로가기 버튼 비노출, 그 외 페이지에서는 노출

### FR-02 메인 홈
- 히어로, 긴급 대응, 성공사례, 변호사 경력, 전문분야 순서로 구성
- 홈 진입 시 데모 설명 모달 표시

### FR-03 법무법인 소개
- 법인 소개 문구
- 변호사 프로필 목록
- 오시는 길/연락처
- 연혁 타임라인 (추가 구현 필요)

### FR-04 전문분야 상세
- 성범죄/마약 각 상세 페이지
- 위험 포인트, 대응 절차, FAQ
- 판례/관련 사례 블록 (추가 구현 필요)

### FR-05 성공사례
- 목록 검색/카테고리 필터
- 상세 페이지(개요/전략/담당자)
- CMS 데이터 연동 CRUD (추가 구현 필요)

### FR-06 온라인 상담
- 필수 입력 검증
- 제출 완료 피드백

### FR-07 CMS
- 로컬스토리지 기반 목록/등록/수정/삭제/리셋
- 현재 일반 콘텐츠 타입 기반
- 성공사례 타입 확장 필요

## 7. 수용 기준 (Acceptance Criteria)

### AC-2-1 소개
- /about에서 법무법인 소개/변호사 소개/오시는 길/연혁 4요소가 모두 보인다.

### AC-2-2 전문분야 상세
- 성범죄/마약 상세 각각에 법률 정보, 사건 유형, 대응 절차, 판례 블록이 존재한다.

### AC-2-3 성공사례 관리
- 관리자가 성공사례를 등록/수정/삭제하면 /cases에 즉시 반영된다.

### AC-2-4 온라인 상담
- 필수값 누락 시 제출 불가, 정상 제출 시 접수 메시지 노출.

### AC-2-5 CMS
- 브라우저 재접속 후에도 로컬스토리지 데이터 유지.
- 데이터 리셋 시 샘플 데이터로 복원.

### 공통 품질 기준
- npm run lint 통과
- npm run build 통과
- 모바일(360px+)에서 주요 화면 레이아웃 유지

## 8. 구현 우선순위

### P0 (즉시 반영)
- 소개 페이지 연혁 섹션 추가
- 전문분야 상세 판례/관련 성공사례 섹션 추가
- 성공사례 CMS CRUD와 /cases 데이터 연동

### P1
- CMS 타입 분리(about/practice/case)
- 입력 유효성 강화 및 에러 상태 UX 개선

### P2
- 서버 저장소 전환(Supabase 또는 Headless CMS)
- 관리자 인증/권한
