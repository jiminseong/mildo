# 법무법인 홈페이지 제안용 데모 앱 작업 계획

기준 문서: `OVERVIEW.md`

목표:
- 공고 내용을 읽고 이해한 수준을 넘어서, 법무법인 홈페이지의 방향성과 상담 전환 구조를 별도 URL로 바로 보여줄 수 있는 제안용 데모를 만든다.
- 법무법인 소개, 전문분야 상세, 성공사례 아카이브, 온라인 상담, CMS 운영 구조를 1개의 흐름 안에서 빠르게 설명한다.
- `mildo` 메인 앱과 분리된 독립 Next.js 앱으로 관리해 제안용 배포와 보관을 쉽게 만든다.

구현 위치:
- 앱 루트: `wishket/법무법인홈페이지신규구축/proposal-demo`
- 화면 코드: `proposal-demo/src/app/*`
- 제안 문서: `APPLICATION.md`, `TASK.md`

생성 기준:
- `npx create-next-app@latest proposal-demo --ts --eslint --tailwind --app --src-dir --import-alias "@/*" --use-npm --yes`
- 배포 대상은 별도 Vercel 프로젝트로 관리한다.
- 메인 서비스 앱 라우트와 연결하지 않고, mock 데이터 중심의 제안용 데모로 유지한다.

데모 범위:
- 첫 화면에서 법무법인의 전문성과 상담 CTA를 함께 보여주는 홈페이지 메인 구조
- 성범죄, 마약 분야를 예시로 한 전문분야 상세 페이지 구성
- 카테고리, 결과, 요약, 담당 변호사 기준으로 탐색 가능한 성공사례 아카이브
- 빠른 상담 신청 흐름과 신뢰 형성용 보조 콘텐츠 구성
- 성공사례/전문분야/상담 내역을 관리하는 CMS 운영 화면 예시
- 모바일에서도 구조가 무너지지 않는 반응형 레이아웃

비범위:
- 실제 DB 연동
- 실제 개인정보 저장 및 보안 정책 구현
- 실제 이메일/SMS 알림 연동
- 전체 기존 사이트 콘텐츠 이관
- 다국어, 통계 대시보드, CRM 연동
- 계약 전 전체 관리자 기능 완성

배포 메모:
- 로컬 실행: `npm run dev`
- 사전 검증: `npm run lint`, `npm run build`
- Vercel에서는 `wishket/법무법인홈페이지신규구축/proposal-demo`를 루트 디렉토리로 연결한다.
- 외부 공유 시에는 완성 서비스가 아니라 `제안용 예시 데모`라는 점을 명확히 표기한다.

디자인 방향:
- 법률 서비스 특성에 맞게 차분하지만 답답하지 않은 톤으로 구성
- 종이 질감에 가까운 배경과 딥 네이비, 브론즈 계열 포인트로 신뢰감 있는 인상 설계
- 정보량이 많은 업종 특성상 장식보다 정보 구조와 CTA 우선순위를 더 명확히 노출
- 성공사례는 게시판보다는 탐색형 아카이브처럼 보이게 구성
- 관리자 화면은 실제 운영자가 어떤 데이터를 만지는지 바로 이해되도록 단순하고 명확하게 표현
