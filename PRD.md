# PRD: 밀도(공식 표기: 밀도 소프트웨어) 외주 홍보용 사이트

## 1. 제품 개요
### 1.1 목적
- **신뢰 구축**: 외주를 “개인 프리랜서”가 아니라 팀/회사처럼 신뢰되게 보여주는 홍보용 웹사이트를 구축합니다.
- **방문자가 30초 안에 아래를 이해하도록 함**:
  - 무엇을 해주는지(자영업 패키지 vs 고도 개발)
  - 결과물/진행 방식(프로세스, 산출물, 운영/유지보수)
  - 바로 문의할 수 있는 경로(폼/카톡/전화/예약)

### 1.2 브랜드/표기 규칙(필수)
- **대표 브랜드(상단 로고/메뉴)**: 밀도
- **공식 표기(푸터/OG/링크드인/견적서)**: 밀도 소프트웨어
- **문장형 소개 1줄(사이트 전역 공용)**:
  “밀도는 자영업자 웹·예약 패키지와 고도 개발을 함께 하는 소프트웨어 파트너입니다.”
- **배경**: “밀도”는 이미 타 업종에서 사용 사례가 있어(검색 혼선 가능), 공식 표기를 고정해 구분성을 확보합니다.

---

## 2. 타깃 사용자
### 2.1 자영업자(로컬 패키지)
- **업종**: 식당/카페/헬스장/미용/학원 등
- **핵심 니즈**: 홈페이지 정리, 지도/전화/카톡 문의 연결, 예약·상담 흐름, 홍보용 랜딩
- **의사결정 포인트**: 비용/납기/유지보수/“내가 이해할 수 있는 설명”

### 2.2 고도 개발(프로덕트/업무 시스템)
- **대상**: 소규모 사업자, 초기 스타트업, 사내 신규 프로젝트 담당자
- **핵심 니즈**: MVP 제작, 관리자/결제/인증/자동화, 기존 서비스 개선/유지보수
- **의사결정 포인트**: 일정·범위 관리, 커뮤니케이션, 운영 안정성, 문서/테스트/배포

---

## 3. 목표(KPI)
### 3.1 1차 KPI(리드)
- 문의 제출 전환율(form_submit / sessions)
- 상담 예약 전환율(가능 시 schedule_complete)
- 문의 품질(예산/일정/요구사항 포함 비율)

### 3.2 2차 KPI(신뢰/관심)
- 포트폴리오 상세 조회율
- 패키지/프로세스 섹션 도달률
- FAQ 열람률 / 체류 시간

---

## 4. 사이트 구조(IA)
### 4.1 공통(밀도 루트)
- **Home**: 메인 페이지
- **서비스(2갈래 분기)**:
  - 자영업 패키지
  - 고도 개발
- **포트폴리오**: 리스트/상세
- **진행 방식**: 프로세스/산출물/유지보수
- **FAQ**
- **문의하기**

### 4.2 SEO용 확장(v1)
- **업종별 랜딩**: 식당 홈페이지 제작, 헬스장 예약 페이지, 카카오채널 문의 랜딩 등

---

## 5. 핵심 사용자 흐름
1. **유입**: 검색/링크드인/지인 소개/커뮤니티
2. **선택**: Home에서 즉시 2갈래 선택(자영업 vs 고도 개발)
3. **탐색**: 각 서비스 페이지에서 “대표 사례 2~3개 + 범위/가격 감 + 프로세스/보증” 확인
4. **전환**: 문의(폼 제출) 또는 상담 예약
5. **확정**: 자동 회신(요약 + 준비자료) → 1차 컨택

---

## 6. 기능 요구사항
### 6.1 MVP 필수 기능
#### A) 콘텐츠/랜딩
- **반응형**: 모바일 우선
- **첫 화면(히어로)**: 한 문장 가치 제안 + CTA 2개(자영업 / 고도 개발)
- **신뢰 섹션(공통)**:
  - 진행 단계(요구사항 → 견적/일정 → 제작 → 검수 → 오픈 → 운영)
  - 산출물 명시(소스/배포/계정/운영가이드 등)
  - 유지보수 정책(무상/유상 범위)

#### B) 포트폴리오
- **리스트**: 업종/유형 필터(선택)
- **상세 템플릿(고정 구조)**:
  문제/목표 → 해결 → 구현 범위 → 결과(정성/정량) → 작업 기간 → 사용 기술 → 맡은 역할

#### C) 문의 리드 수집
- **문의 폼(서비스 선택 포함)**:
- **자영업**: 업종/주소(지도)/예약 필요 여부/참고 사이트/희망 납기/예산 범위
- **고도 개발**: 현재 단계(MVP/운영)/필요 기능 체크리스트/기존 코드 유무/희망 일정/예산 범위
- **제출 후**:
  - Thank-you 페이지 + “다음 단계(자료/일정)” 안내
  - 자동 회신 이메일(요약 + 준비자료 체크리스트)

#### D) 연락 채널(보조 전환)
- 전화 / 카카오채널 / 이메일 링크(상단 고정 또는 문의 영역에 배치)

### 6.2 v1 확장 기능
- 업종별 SEO 랜딩(10개 내외부터 시작)
- 간단 견적 범위 제시(패키지/옵션 선택 → “예상 범위”)
- 후기 수집 폼 + 승인 후 노출
- 블로그(사례/가이드)로 검색 유입 확대

---

## 7. GA4(구글 애널리틱스) 요구사항
### 7.1 설치/운영 원칙
- GA4 기본 설치 + 권장: GTM으로 이벤트 관리
- 내부 트래픽 제외(개발/본인 기기)
- UTM 규칙 운영

### 7.2 이벤트 설계(필수)
- `service_select` : params: service(local|advanced), source_page
- `cta_click` : params: cta_id, service, page_type
- `form_start` / `form_submit` : params: form_id, service, budget_range, timeline
- `portfolio_view_detail` : params: portfolio_id, industry, service_guess
- `faq_expand` : params: faq_id, service
- `contact_click` : params: method(phone|kakao|email), service

### 7.3 전환(Conversions)
- 1순위: `form_submit`
- 2순위: `schedule_complete`
- 보조: `contact_click`

### 7.4 리포트(주간 체크)
- 채널별 전환 / 서비스별 전환 / 페이지별 기여 / 폼 퍼널

## 8. SEO 요구사항
### 8.1 기본 SEO
- 페이지별 Title/Meta, OG, sitemap/robots
- 포트폴리오 상세에 구조화된 정보

### 8.2 전략
- “밀도” 단독 검색보다는 “업종/의도 키워드 랜딩”으로 유입 확보
  - 예: “고깃집 홈페이지 제작”, “헬스장 예약 페이지”

## 9. 디자인 가이드
### 9.1 공통 톤
- **목표**: 정돈감(타이포/여백/정보 구조)으로 신뢰 구축
- **금지**: 네온/글로우/과한 그라데이션/3D 블롭

### 9.2 Typography
- **폰트**: Pretendard
- **원칙**: 본문 가독성(줄간격 1.6), 강조는 1단계만

### 9.3 Color(아이보리 톤 통일)
- **배경**: 아이보리 계열
- **텍스트**: 완전 블랙보다 한 톤 부드럽게
- **구분선/카드**: 대비 낮게, 여백으로 구분

### 9.4 서비스별 무드
- **자영업 패키지**: 북유럽 느낌, 따뜻하고 담백, 실용적
- **고도 개발**: 고급/정제, 타이포 중심, 표/리스트 활용

### 9.5 “AI 티 안 나게” 체크리스트
- 뭉술한 문구 → 구체적 산출물/기간으로 대체
- 스톡 이미지 → 실제 캡처/도식으로 대체
- 스타일 통일성 체크
