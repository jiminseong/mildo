기준일: 2026-03-13  
대상 프로젝트: `AIM 앱 패밀리`  
구성 범위: `AIM Productivity` + `AIM Wellness` + `AIM 랜딩 페이지`

이 문서는 외주 포트폴리오 등록 화면에 바로 붙여넣을 수 있도록 `AIM 앱 패밀리` 기준으로 재구성한 통합 초안이다.

## 2. 프로젝트 배경

### 복붙용 문안

```text
1) 문제점
- 생산성 기능과 웰니스 기능을 하나의 서비스로 운영하면 사용자 목적이 달라 핵심 가치가 흐려질 수 있었습니다.
- App Store 메타데이터, 스크린샷, 심사 설명도 앱별로 분리하지 않으면 메시지 충돌과 운영 리스크가 커졌습니다.
- 브랜드는 하나로 유지하면서도 사용 목적에 따라 다른 경험을 제공할 수 있는 제품 구조가 필요했습니다.

2) 프로젝트 목표
- 하나의 공통 코드베이스를 유지하면서 Productivity와 Wellness를 별도 앱처럼 운영할 수 있는 variant 구조를 구축합니다.
- 공개 랜딩 페이지를 통해 두 앱의 가치와 다운로드 동선을 명확하게 안내합니다.
- 제품 기획, 디자인, 개발, 배포, 심사 대응까지 일관된 운영 체계를 만듭니다.

3) 주안점
- 공통 인프라는 재사용하되 제품 범위와 메시지는 앱별로 분리
- 각 앱의 핵심 기능을 선명하게 보여주는 IA와 스토어 구조 설계
- 배포 이후에도 유지 가능한 문서화와 운영 프로세스 구축
```

## 3. 프로젝트 상세

### 복붙용 문안

```text
1) 프로젝트 소개
- AIM 앱 패밀리는 하나의 브랜드 아래에서 Productivity 앱, Wellness 앱, 공개 랜딩 페이지를 함께 운영하는 멀티 제품 구조입니다.
- 생산성 관리와 건강 관리의 사용 목적이 다르다는 점에 주목해, 하나의 통합 앱이 아니라 목적별로 분리된 두 개의 앱과 이를 연결하는 공개 웹을 설계했습니다.

2) 담당 범위
- 서비스 전략 수립, 정보 구조 설계, UI/UX 디자인, 프론트엔드 개발, Supabase 백엔드 연동, App Store 배포 및 심사 대응까지 전 과정을 담당했습니다.
- 공통 브랜드 경험은 유지하면서도 Productivity와 Wellness가 각자 다른 문제를 해결하도록 제품 구조와 운영 기준을 함께 설계했습니다.

3) 제품 분리 전략
- AIM Productivity는 Todo, Routine, Mandalart 중심의 실행 관리 앱으로 설계했습니다.
- AIM Wellness는 Workout, Nutrition, Health Profile 중심의 운동·영양 관리 앱으로 설계했습니다.
- 공개 랜딩 페이지는 두 앱의 핵심 가치, 다운로드 CTA, 정책 페이지, 프로덕트 노트를 연결하는 브랜드 허브 역할로 구성했습니다.

4) 기술 구현 포인트
- 하나의 모노레포와 공통 런타임 위에서 `unified`, `productivity`, `wellness` 3개 variant 구조를 운영했습니다.
- React Native + Expo Router 기반 앱과 Next.js 기반 랜딩 페이지를 함께 구성하고, Supabase Auth, PostgreSQL, RLS, Edge Function으로 데이터와 인증을 관리했습니다.
- Variant Guard, 공통 쿼리 키 네임스페이스, 앱별 PRD/마케팅 문서 분리를 통해 제품 범위와 운영 기준이 섞이지 않도록 설계했습니다.

5) 주요 구현 내용
- Productivity에는 날짜/리스트/달력 기반 Todo, 반복 루틴, 9x9 Mandalart 기능을 구현했습니다.
- Wellness에는 운동 세션 기록, 3대 운동 추이, 시간대별 영양 체크, 건강 프로필 기능을 구현했습니다.
- 랜딩 페이지에는 두 앱 소개, 각 App Store 다운로드 CTA, 블로그, 개인정보 처리방침, 이용약관 페이지를 구축했습니다.

6) 출시 및 운영
- App Store 메타데이터와 스크린샷, 심사 메모를 앱별로 분리 관리하는 운영 체계를 만들었습니다.
- Productivity와 Wellness 각각의 iOS 심사 이슈를 대응했고, Productivity macOS Electron/MAS 배포 파이프라인까지 정리했습니다.
- 현재도 두 앱의 안정화와 기능 고도화, 공개 웹 운영을 지속하고 있습니다.
```

## 8. 포트폴리오 입력 시 주의사항

- 이 포트폴리오는 `AIM Wellness 단일 앱`이 아니라 `AIM 앱 패밀리 통합 구축 사례`로 보는 편이 맞다.
- 핵심 메시지는 `하나의 브랜드/코드베이스에서 제품 목적에 따라 2개 앱과 공개 웹을 분리 운영했다`는 점이다.
- `AI 코칭 상용화 완료` 같은 과장 표현보다는 `공통 인프라 위에서 제품 분리와 출시 운영 체계를 구현했다`는 표현이 안전하다.
- 실사용자 수, 매출, 리텐션 등 외부 성과 수치는 현재 문서 근거가 없으므로 넣지 않는 것을 권장한다.
- 성과 항목은 `variant 구조`, `앱 분리 전략`, `스토어 운영`, `Electron/MAS`, `랜딩 구축`처럼 검증 가능한 사실 위주로 유지한다.
