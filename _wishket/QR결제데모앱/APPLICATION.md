# QR 결제 데모 앱 개발 지원서 초안

## 기본 입력값

- 지원 금액: 2,500,000원
- 지원 기간: 30일
- 관련 포트폴리오: React Native 앱 2개 운영, 현장용 앱 + 관리자 시스템 + 결제/운영 상태 흐름 경험 보유

### 금액/기간 제안 메모

- 공고 범위는 실결제 구축이 아니라, 정책자금 신청 및 투자 발표용 QR 결제 시나리오를 안정적으로 재현하는 데모 앱 + 관리자 페이지 구축입니다.
- 따라서 PG/알리페이/위챗페이 연동, 금융 보안 인증, 정산 로직을 제외하고 `사용자 결제 시나리오 + 가맹점 승인 + 관리자 조회/초기화`에 집중하면 구현 범위를 명확히 통제할 수 있습니다.
- 금액을 공격적으로 낮춘 이유는 난이도를 가볍게 본 것이 아니라, React Native/관리자/상태 기반 흐름 경험이 있어 1차 데모 범위를 빠르게 압축할 수 있기 때문입니다.
- 발표용 데모는 기능 수보다 `시연자가 원하는 순서대로 항상 같은 결과가 나오는가`가 더 중요하므로, 계정/시드데이터/거래 초기화 같은 시연 안정성까지 포함한 구조로 제안하는 편이 낫습니다.
- 중국어 UI는 적용 가능하되, 전문 번역 감수는 별도 협의로 분리하는 것이 안전합니다.

## 작성 메모

- 가격을 낮추더라도 `포트폴리오가 필요해서 싸게 한다`가 아니라, `실결제 제외된 1차 데모 범위를 명확히 잘라서 공격적으로 제안한다`는 구조로 정리했습니다.
- 우대사항 3개는 단순 체크가 아니라, 각각을 이번 공고의 핵심 문제인 `QR 상태 전이`, `시연 안정성`, `중국어 UI 적용 방식`으로 연결했습니다.
- 특히 데모 프로젝트에서 중요한 `시연 계정`, `시드 데이터`, `상태 복원` 포인트를 앞에 넣어, 단순 앱 개발자보다 발표/검증용 MVP를 이해하는 사람처럼 보이게 조정했습니다.
- 250만 원은 매우 공격적인 금액이므로, 지원서 톤은 반드시 `실결제 제외 + 데모 범위 한정 + 시연 안정성 중심`으로 유지하는 편이 좋습니다.
