# QR 결제 데모 앱 개발 지원서 초안

## 기본 입력값

- 지원 금액: 4,000,000원
- 지원 기간: 30일
- 관련 포트폴리오: React Native 앱 2개 운영, 포토부스 결제/운영 흐름 프로젝트 경험 보유

### 금액/기간 제안 메모

- 본 공고는 실제 결제 서비스 구축이 아니라, 정책자금 신청 및 투자 발표용으로 QR 결제 핵심 흐름을 시연하는 데모 앱 개발 과업으로 판단했습니다.
- 따라서 실제 PG 연동, 알리페이/위챗페이 연동, 금융 보안 인증을 제외하고 `사용자 결제 흐름 + 가맹점 승인 + 관리자 조회`에 집중하면 공고 예산 4,000,000원과 30일 일정 안에서 충분히 납품 가능한 범위입니다.
- React Native 앱 2개를 별도 제품처럼 설계·배포·운영 중인 경험이 있어, 이번 건도 Flutter보다 React Native 기준으로 제안하는 편이 강점과 직접 연결됩니다.
- 일정 안정성을 높이기 위해 1차는 Android 앱 1종 안에서 사용자 모드와 가맹점 모드를 분리하고, 관리자 페이지는 Firebase 기반의 기본 CRUD/조회 중심으로 제안합니다.
- 중국어 전문 번역 감수나 실제 결제 연동은 본 제안 범위 밖으로 분리하는 것이 리스크 관리에 유리합니다.

## 제출용 본문

```text
지원 금액: 4,000,000원
지원 기간: 30일

- React Native 기반 앱을 실제로 2개 variant로 분리 운영한 경험이 있어, 이번 공고처럼 모바일 앱 중심의 핵심 흐름을 빠르게 정리해야 하는 프로젝트에 바로 적용 가능합니다.
- 또한 비공개 포토부스 운영 프로젝트에서 현장용 앱과 관리자 시스템을 함께 설계·구축하며, 결제 전후 상태 변화와 운영자 확인 흐름이 맞물리는 구조를 다뤄본 경험이 있습니다.
- 본 프로젝트 역시 실제 결제망 구축보다 `QR 스캔 -> 금액 입력 -> 비밀번호 입력 -> 결제 완료`, `가맹점 승인`, `관리자 거래 조회`가 한 번에 이해되도록 만드는 것이 핵심이라고 판단했습니다.

<유사 프로젝트 진행 경험>
- AIM 앱 패밀리에서는 하나의 브랜드 아래 React Native 앱 2개(Productivity, Wellness)를 별도 제품처럼 설계·개발·배포·운영하고 있습니다.
- 공통 코드베이스를 유지하면서도 앱별 기능, 스토어 메타데이터, 배포 흐름, 운영 기준을 분리해 관리한 경험이 있어, 이번 프로젝트처럼 Android 앱을 짧은 기간 안에 안정적으로 정리하는 작업과 잘 맞습니다.
- 비공개 포토부스 운영 프로젝트에서는 현장용 앱과 관리자 시스템을 함께 구축했고, 운영자가 설정과 상태를 직접 관리할 수 있는 구조를 설계했습니다.
- QR 결제 로직이 필요한 프로젝트에서도 중요한 것은 실결제 연동 자체보다, 결제 요청 생성, 승인 상태 반영, 완료 결과 노출, 거래 내역 누적처럼 상태 전이가 끊기지 않게 설계하는 것인데, 이 부분을 실무 프로젝트에서 계속 다뤄왔습니다.

<프로젝트 진행 제안>
- 앱은 React Native 기준으로 제안드립니다. Android APK 납품 목적에 맞게 사용자/가맹점 역할을 한 앱 안에서 분리하고, 관리자 페이지는 Firebase 기반으로 구성하는 방식이 가장 효율적입니다.
- 사용자 앱에는 간편 로그인, 잔액 조회, 고정 금액 충전, QR 스캔, 결제 금액 입력, 결제 비밀번호 입력, 결제 완료 화면, 거래 내역 조회를 포함합니다.
- 가맹점 기능에는 고유 QR 코드 표시, 결제 요청 확인, 승인/거절 mock 처리, 일/주/월 매출 내역 조회를 포함합니다.
- 관리자 페이지에는 데모 사용자 계정 관리, 테스트 가맹점 등록/조회, 전체 거래 내역 조회 및 검색 기능을 포함합니다.
- 모든 UI 텍스트는 중국어 기준으로 적용하되, 전문 번역 감수가 필요한 경우에는 별도 협의 범위로 나누는 것이 안전합니다.

[예상 일정]
- 1주차: 요구사항 정리, 시연 시나리오 확정, 화면/데이터 구조 설계
- 2주차: 사용자 앱 핵심 흐름 구현(로그인, 잔액, 충전, QR 결제)
- 3주차: 가맹점 승인 기능 및 관리자 페이지 구현, Firebase 데이터 연동
- 4주차: QA, 중국어 문구 최종 반영, APK 빌드, 시연 계정 및 관리자 정보 정리
- 실제 결제망과 금융 보안 인증을 제외하고 핵심 시연 플로우에 집중하는 범위이므로, 30일 안에 안정적으로 완수 가능하다고 판단하여 제안드립니다.

- 최종 산출물: Android APK, 앱 및 관리자 전체 소스코드, 관리자 접속 정보
```

## 작성 메모

- 공고 핵심을 `실결제 구축`이 아니라 `시연용 흐름 구현`으로 정확히 잡아 일정과 예산의 설득력을 확보했습니다.
- 제안 범위는 Android 앱 1종(사용자/가맹점 역할 분리) + 관리자 페이지로 묶어, 예산 4,000,000원 안에서 과하지 않게 납품 가능한 범위로 정리했습니다.
- 유사 경험 블록을 본문 상단으로 끌어올려 React Native 앱 2개 운영 경험과 포토부스 운영/결제 흐름 경험이 첫 화면에서 보이도록 조정했습니다.
