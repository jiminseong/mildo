# QR 결제 데모 앱 작업 계획

기준 문서: OVERVIEW.md

## 1. 목표

- 정책자금 신청 및 투자 발표용으로 QR 결제 핵심 흐름을 명확히 시연할 수 있는 Android 데모 앱과 관리자 페이지를 구축한다.
- 실제 결제망 연동 없이도 사용자, 가맹점, 관리자 역할별 흐름이 자연스럽게 이어지도록 mock 상태를 설계한다.
- 30일 내 APK 납품, 관리자 접속 환경 정리, 소스코드 인계가 가능한 범위로 기능을 통제한다.

## 2. 구현 방향

- Android 앱 1종
  - 데모 로그인 후 `사용자 모드`와 `가맹점 모드`를 분리 운영
  - QR 스캔, 결제 요청, 승인, 내역 조회까지 단일 앱 안에서 시연 가능하게 구성
- 관리자 웹 1종
  - Firebase 기반으로 데모 사용자/가맹점/거래 내역 CRUD 및 조회 화면 구성
- 백엔드/데이터
  - Firebase Auth, Firestore 기준
  - 가맹점 승인 알림은 Firebase 실시간 동기화 또는 단순 polling 구조로 구현
- 언어
  - 앱/관리자 UI 텍스트는 중국어 기준으로 구성

## 3. 1차 구축 범위

### 3.1 사용자 앱 기능

- 간편 로그인
- 가상 잔액 조회
- 고정 금액 충전
- 카메라 기반 QR 스캔
- 결제 금액 입력
- 결제 비밀번호 입력
- 결제 완료 mock 화면
- 충전/결제 거래 내역 조회

### 3.2 가맹점 기능

- 가맹점 고유 QR 코드 생성 및 노출
- 결제 요청 수신
- 결제 승인/거절 mock 처리
- 일/주/월 매출 내역 조회

### 3.3 관리자 페이지

- 데모 사용자 계정 생성/조회/상태 관리
- 테스트 가맹점 등록/조회/QR 정보 관리
- 전체 거래 내역 조회/검색
- 데모 데이터 초기화 또는 시드 데이터 관리 기능(협의)

## 4. 비범위

- 실제 PG, 알리페이, 위챗페이 연동
- 금융 보안 인증, 실명 인증, 법적 결제 요건 대응
- 정산, 환불, 부분 취소, 세금계산서 처리
- iOS 앱 개발
- 운영 수준의 통계 대시보드 및 권한 체계 고도화
- 중국어 전문 번역 감수
