# QR 결제 데모 앱 PRD

## 1. 문서 개요

- 프로젝트명: QR 결제 데모 앱 개발
- 개발 기간: 30일
- 지원 디바이스: Android 앱 + 관리자 웹
- 목표: 실제 결제 시스템 없이 QR 결제 핵심 흐름을 시각적으로 시연 가능한 수준으로 구현

## 2. 배경 및 목표

- 본 프로젝트의 목적은 정책자금 신청 및 투자 유치 발표에서 사업 모델의 핵심인 QR 결제 구조를 직관적으로 보여주는 것이다.
- 실제 알리페이/위챗페이 연동이나 금융 보안 인증은 본 범위가 아니므로, 사용자가 체감하는 결제 흐름과 가맹점 승인 경험을 mock 기반으로 재현하는 것이 핵심이다.
- 시연자는 하나의 데모 환경 안에서 사용자 결제, 가맹점 승인, 관리자 거래 조회까지 연속된 스토리로 설명할 수 있어야 한다.

## 3. 과업 범위

### 3.1 포함 범위 (MVP)

- Android 데모 앱
  - 사용자 로그인
  - 잔액 조회
  - 가상 충전
  - QR 스캔
  - 결제 금액 입력
  - 결제 비밀번호 입력
  - 결제 완료 화면
  - 거래 내역 조회
- 가맹점 기능
  - 가맹점 QR 코드 생성/표시
  - 결제 요청 수신
  - 결제 승인/거절 mock 처리
  - 매출 내역 조회
- 관리자 웹
  - 사용자 관리
  - 가맹점 관리
  - 거래 내역 조회/검색
- 백엔드/데이터
  - Firebase 기반 데이터 저장 및 상태 동기화

### 3.2 제외 범위 (후속)

- 실제 결제 연동
- 알리페이/위챗페이 직접 연동
- 금융 보안 인증, 법적 인증 절차
- 환불/부분 취소/정산 로직
- iOS 앱 지원
- 고급 권한 시스템 및 운영 통계 대시보드

## 4. 사용자 유형

- 사용자: 잔액 충전 후 QR 결제를 시도하는 일반 이용자 역할
- 가맹점: 고유 QR을 제시하고 결제 요청을 승인하는 점주 역할
- 관리자: 데모 계정과 거래 흐름을 관리/조회하는 운영자 역할
- 발표자: 위 3개 역할을 상황에 따라 전환하며 시연하는 사용자

## 5. 핵심 사용자 흐름

1. 사용자
   로그인 -> 잔액 확인 -> 충전 -> QR 스캔 -> 결제 금액 입력 -> 결제 비밀번호 입력 -> 가맹점 승인 대기 -> 결제 완료 -> 거래 내역 확인

2. 가맹점
   로그인 -> 내 QR 표시 -> 결제 요청 확인 -> 승인/거절 -> 매출 내역 확인

3. 관리자
   로그인 -> 사용자 계정 생성/조회 -> 가맹점 등록/QR 정보 확인 -> 전체 거래 내역 검색

## 6. 상세 기능 요구사항

### FR-01 데모 로그인

- 사용자/가맹점/관리자 각 역할에 맞는 데모 계정으로 간편 로그인 가능해야 한다.
- 실제 회원가입 대신 사전 생성 계정 또는 간단한 인증 정보로 진입한다.

### FR-02 잔액 및 충전

- 로그인 후 현재 가상 잔액을 즉시 확인할 수 있어야 한다.
- 고정 금액 버튼으로 가상 충전을 실행할 수 있어야 한다.
- 충전 완료 후 잔액과 거래 내역이 즉시 반영되어야 한다.

### FR-03 QR 스캔 및 결제 요청

- 사용자 앱에서 카메라로 QR 코드를 스캔할 수 있어야 한다.
- QR에는 최소한 가맹점 식별 정보가 포함되어야 한다.
- 사용자는 결제 금액을 직접 입력할 수 있어야 한다.

### FR-04 결제 비밀번호 및 완료 화면

- 사용자는 결제 직전 비밀번호를 입력해야 한다.
- 비밀번호 확인 후 결제 요청이 생성되어야 한다.
- 승인 완료 시 결제 완료 화면과 최신 잔액이 표시되어야 한다.

### FR-05 가맹점 QR 생성/표시

- 가맹점은 본인 식별 정보를 기반으로 생성된 QR을 화면에 표시할 수 있어야 한다.
- QR 정보는 관리자에서 수정 가능하거나 앱 내 고정 값으로 관리할 수 있어야 한다.

### FR-06 가맹점 승인 mock

- 사용자 결제 요청이 발생하면 가맹점에게 요청 목록 또는 알림 형태로 노출되어야 한다.
- 가맹점은 승인/거절을 선택할 수 있어야 한다.
- 승인 결과는 사용자 앱과 관리자 거래 내역에 반영되어야 한다.

### FR-07 거래/매출 내역

- 사용자는 충전/결제 내역을 시간순으로 조회할 수 있어야 한다.
- 가맹점은 일/주/월 기준 매출 요약과 목록을 조회할 수 있어야 한다.
- 관리자는 전체 거래를 사용자/가맹점/상태 기준으로 검색할 수 있어야 한다.

### FR-08 관리자 기능

- 관리자는 데모 사용자 계정을 생성/조회/수정/비활성화할 수 있어야 한다.
- 관리자는 가맹점 정보를 등록/조회/수정할 수 있어야 한다.
- 관리자는 전체 거래 내역을 검색하고 상태를 확인할 수 있어야 한다.

### FR-09 중국어 UI

- 사용자에게 노출되는 앱 UI 텍스트는 중국어 기준으로 제공되어야 한다.
- 관리자 페이지는 중국어 또는 운영 편의상 한국어/중국어 혼합 여부를 사전 합의한다.

## 9. 비기능 요구사항

- Android 카메라 권한 요청 및 QR 스캔 동작이 정상 작동해야 한다.
- 데모 계정 기준으로 주요 화면 진입이 빠르고 안정적으로 동작해야 한다.
- 앱/관리자 데이터는 Firebase 기준으로 동일한 상태를 보여야 한다.
- 중국어 텍스트가 UI 깨짐 없이 표시되어야 한다.
- 최종 산출물로 APK와 관리자 접속 정보가 제공되어야 한다.

## 10. 수용 기준

1. 데모 계정으로 사용자/가맹점/관리자 진입이 가능해야 한다.
2. 사용자 충전 후 잔액이 즉시 반영되어야 한다.
3. 사용자 QR 결제 요청 후 가맹점 승인 흐름이 동작해야 한다.
4. 승인 완료 시 사용자 거래 내역, 가맹점 매출 내역, 관리자 거래 내역에 동일 거래가 반영되어야 한다.
5. 관리자 페이지에서 사용자/가맹점/거래 내역 조회가 가능해야 한다.
6. 앱 주요 UI가 중국어 기준으로 구성되어야 한다.
7. Android APK 빌드 및 전달이 가능해야 한다.
