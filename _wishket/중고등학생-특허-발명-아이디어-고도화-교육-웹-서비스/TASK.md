# 제안용 독립 데모 앱 작업 계획

기준 문서:
- `OVERVIEW.md`
- `PRD.md`

목표:
- 공고 내용을 읽고 이해한 수준을 넘어, 학생과 운영자가 실제로 눌러보며 전체 흐름을 시연할 수 있는 프론트엔드 전용 프로토타입을 만든다.
- 백엔드 없이도 로그인, 개인정보 입력, 발명가 번호 발급, 진단 분기, 카드 선택, 익명 피드백, 관리자 승인, 최종 퀴즈까지 전 기능을 mock 상태로 구현한다.
- `mildo` 메인 앱 라우트에 섞지 않고, 별도 Next.js 앱으로 분리해 독립 배포한다.

구현 위치:
- 앱 루트: `wishket/중고등학생-특허-발명-아이디어-고도화-교육-웹-서비스/idea-workshop-demo`
- 화면 코드: `idea-workshop-demo/src/app/*`
- 문서 기준: `PRD.md`

생성 기준:
- `npx create-next-app@latest idea-workshop-demo --ts --eslint --tailwind --app --src-dir --import-alias "@/*" --use-npm --yes`
- 배포 대상은 별도 Vercel 프로젝트로 관리한다.
- 루트 서비스와 코드/배포를 분리하고, 제안 종료 후에도 보관 또는 폐기 판단이 쉽도록 독립 패키지로 둔다.
- 한글 경로 아래 앱을 둘 경우, `dev`/`build` 스크립트는 `--webpack` 기준으로 둔다.

데모 범위:
- 이름 + 접속코드 기반 mock 로그인
- 개인정보 입력 및 발명가 번호 발급
- 튜토리얼/인트로 재생형 화면
- 진단 분기와 추천 아이디어 제안
- 15단계 기준 아이디어 고도화 플로우
- 카드 기반 선택과 선택 결과 요약
- 발명가 번호 기반 익명 게시판, 필터, 피드백 작성
- 받은 질문/피드백 답변 및 수정 메모
- Level 2 / Level 3 제출과 관리자 승인 대기
- 운영자 콘솔의 학생 데이터 조회 및 승인/보완 요청
- 최종 아웃트로/퀴즈/완료 화면
- 우측 상단 가이드 팝업
- 모바일에서도 무너지지 않는 반응형 구성

비범위:
- 실제 OTP 발급
- 실제 DB 연동
- 실제 관리자 인증 서버
- 실제 파일 저장소 업로드
- 실제 통계 엔진 연동
- 제안용 앱과 메인 서비스의 공통 컴포넌트 통합

배포 메모:
- 로컬 확인: `npm run dev`
- 배포 전 확인: `npm run lint`, `npm run build`
- Vercel에서는 `idea-workshop-demo` 디렉토리를 루트로 연결한다.
- 댓글/메시지에는 완성 서비스가 아니라 `제안용 예시 프로토타입`이라는 점을 분명히 적는다.

디자인 방향:
- 교육 서비스와 워크숍 도구의 중간 톤으로, 교실용 제품처럼 따뜻하지만 값싸 보이지 않게 구성
- 실물 교보재가 디지털 워크스페이스로 바뀐 느낌이 들도록 종이, 보드, 승인 스탬프 감성을 일부 차용
- 학생 화면과 관리자 화면의 목적 차이가 한눈에 보이도록 앱 셸 구조를 분리
- 설명형 랜딩보다 실제로 사용 중인 프로토타입처럼 보이게 상태 변화와 입력 반응을 우선
