# 골프공 인쇄/각인 키오스크 프로그램 개발 지원서 초안

## 기본 입력값

- 지원 금액: 14,000,000원 (사전 장비 검증 조건부)
- 지원 기간: 60일 (협의 가능)
- 관련 포트폴리오: 키오스크 + 포토 프린터 연동 경험 보유

### 금액/기간 산정 근거

- 본 프로젝트는 Electron 기반 UI 개발 난이도보다, 장비 연동 안정화(프린터/결제기/커팅기/스크린)와 장애 복구 시나리오 검증 비중이 큰 과업입니다.
- 따라서 60일 내 완성도를 확보하려면 초기 2주를 장비 연동 검증(Discovery/PoC)으로 운영하고, 이후 본개발 및 통합 테스트를 진행하는 방식이 현실적입니다.
- 금액은 초기 제안 대비 인하하되, 계약 전 프린터 실물 검증(현장 방문 또는 장비 배송 기반 테스트) 진행을 조건으로 제안하는 것이 상호 리스크 관리에 유리합니다.

## 제출용 본문

```text
<프로젝트 진행 제안>
[유사 프로젝트 경험]
- 포토부스 키오스크 구축 경험이 있으며, DNP 프린터 연동 및 출력 파이프라인(작업 큐/재출력/장비 상태 대응) 구현 경험을 보유하고 있습니다.
- 프린터 연동은 실제 운영 안정성에 직결되는 영역이라, 장비 프로토콜 확인부터 예외 복구 흐름까지 우선 설계하는 방식으로 수행해왔습니다.
- 다수 키오스크 운영 환경에서 관리자 시스템(장비 상태 모니터링, 결제/주문 이력 관리, 장애 로그 조회, 재처리 기능) 구축 경험이 있습니다.
- 원격 운영 시스템(원격 상태 점검, 장애 알림 확인, 운영 설정 반영, 원격 대응 프로세스) 구축/운영 경험이 있어 유지보수 관점까지 포함한 구조 설계가 가능합니다.
- 장비 연동 프로젝트 특성상, 기능 개발과 함께 현장 기반 테스트/운영 안정화까지 포함해 진행 가능합니다.
- 기업 행사/브랜드 이벤트 현장에서 운용된 장비 납품 및 운영 경험이 있어, 실사용 환경 기준의 안정화 포인트를 이해하고 있습니다.

[프로젝트 이해]
- 현재 운영 중인 키오스크 1대의 화면/동작을 기준으로 동일 기능을 신규 구축하고, 기존 하드웨어와 안정적으로 연동되는 윈도우 기반 키오스크 소프트웨어를 만드는 프로젝트로 이해했습니다.
- 핵심은 화면 재현 자체보다, 실제 운영 환경에서 장비 제어 실패/재시도/복구까지 포함한 안정적인 실행 흐름을 구현하는 것입니다.

[수행 전략]
- 1단계(2주): 장비 연동 기술검증(Discovery/PoC)
  - 장비별 통신 방식 확인(USB-Serial, TCP/IP, 벤더 SDK/DLL)
  - 기본 제어 명령 송수신 및 장비 상태 수집 테스트
  - 결제 승인/취소, 인쇄/각인 작업 큐, 오류 코드 수집 검증
- 2단계(4주): 본개발
  - 키오스크 UI/UX 구현 (기존 화면/동작 벤치마킹)
  - 장비 어댑터 계층(프린터/결제/커팅기/스크린) 분리 설계
  - 관리자 기능(상태 모니터링, 결제 내역, 통계, 장애로그) 구현
- 3단계(2주): 통합 테스트 및 안정화
  - 실기기 반복 테스트, 예외 상황 회복 시나리오 점검
  - 운영 가이드 및 소스코드 인수인계

[기술 제안]
- Electron + Node.js 기반 데스크톱 앱
- 장비 연동: 장비별 SDK 또는 시리얼/소켓 통신 모듈화
- 로컬 데이터: SQLite 기반 결제/작업/로그 이력 저장
- 장애 대응: 타임아웃, 재시도, 중복결제 방지, 작업 재개 큐 적용

[금액 제안]
- 총 제안 금액은 14,000,000원으로 제안드립니다.
- 단, 장비 연동 리스크를 줄이기 위해 계약 전 또는 계약 직후 1주 내 프린터 실물 검증 일정을 우선 진행하는 조건을 포함합니다.

[계약 전 장비 검증 제안]
- 프린터 장비는 현장 방문 점검 또는 장비 배송 테스트 중 가능한 방식으로 사전 검증을 진행하면, 이후 개발 일정과 품질 예측 정확도를 크게 높일 수 있습니다.
- 검증 항목은 드라이버/SDK 정상 동작, 출력 파이프라인, 오류 코드 수집, 재출력 및 복구 시나리오까지 포함합니다.
- 프린터 검증이 완료되면 결제기/커팅기 연동 단계로 빠르게 확장해 전체 일정 지연을 최소화할 수 있습니다.

[착수 전 확인이 필요한 핵심 질문]
1) 프린터/결제기/커팅기/스크린의 제조사와 모델명은 무엇인가요? (장비 라벨 사진 가능 여부)
2) 각 장비의 드라이버, SDK, DLL, 샘플 코드, 프로토콜 문서를 제공받을 수 있나요?
3) 커팅기는 독립 장비인가요, 아니면 프린터 내장 오토커터인가요?
4) 인쇄와 각인은 동일 장비에서 처리하나요, 별도 장비(예: UV 프린터 + 레이저 마킹기)로 분리되어 있나요?
5) 결제 단말기 VAN사와 연동 방식(시리얼/LAN/전용 API)은 무엇인가요?
6) 결제 승인/취소/망취소/부분취소 중 어떤 시나리오까지 요구되나요?
7) 기존 프로그램에서 장비 오류 발생 시 현재 운영 복구 절차는 어떻게 되나요?
8) 오프라인/네트워크 장애 시 허용되는 기능 범위는 어디까지인가요?
9) 현장 테스트 가능한 장비 접근 시간/횟수와 원격 접속 가능 여부는 어떻게 되나요?
10) 월 단위 유지보수 범위(장애 대응 SLA, 원격 점검, 기능 개선)는 어떻게 정의할 계획인가요?

[예상 일정]
- 1~2주차: 장비 연동 기술 검증 및 명세 확정
- 3~6주차: 핵심 기능 개발(편집/인쇄/각인/결제/관리자)
- 7~8주차: 통합 QA, 예외 케이스 테스트, 배포 및 인수인계

[산출물]
- 실행 프로그램(윈도우 키오스크)
- 전체 소스 코드 원본(Electron/Node.js)
- 장비 연동 모듈 및 설정 파일
- 관리자 기능(상태/결제/통계) 화면
- 운영 매뉴얼 및 장애 대응 가이드

<커뮤니케이션>
- 위시켓 메시지 + 정기 미팅 기준으로 진행하며, 주 단위 진행 리포트(완료/이슈/다음 주 계획)를 공유하겠습니다.
- 이슈는 재현 절차, 원인, 조치 계획을 문서화해 투명하게 관리하겠습니다.
```

## 작성 메모

- 본 프로젝트는 초기 장비 정보 확보 여부가 일정 리스크를 좌우하므로, 착수 전 질문 항목을 제안서에 명시했습니다.
- 계약은 Discovery/PoC와 본개발을 단계 분리하면 일정/품질 리스크를 크게 줄일 수 있습니다.
