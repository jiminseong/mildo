# 골프공 인쇄/각인 키오스크 데모 디자인 시스템

## 1. 디자인 원칙

- 빠른 선택: 사용자가 3단계 이내로 결제 직전까지 도달하도록 화면을 단순화한다.
- 오류 예방: 장비 상태/오류를 즉시 시각화해 사용자가 혼란 없이 다음 행동을 선택할 수 있게 한다.
- 운영 안정성: 운영자 화면은 화려함보다 가독성과 즉시 대응성을 우선한다.
- 터치 최적화: 키오스크 터치 환경에 맞는 충분한 버튼 크기와 간격을 유지한다.

## 2. 톤앤매너

- 키워드: 신뢰, 또렷함, 산업형 안정감, 직관성
- 카피 톤: 짧고 명령형에 가까운 안내 문구 중심
- 메시지 우선순위: 현재 단계 -> 필요한 행동 -> 문제 발생 시 대안

- Caption: 14/20, 500

## 5. 레이아웃 가이드

- 기준 해상도: 1080x1920 (세로형 키오스크)
- 컨테이너 최대폭: 920px
- 안전 여백: 좌우 40px, 상하 32px
- 터치 타겟 최소 크기: 56x56px
- 주요 간격 단위: 8px 기반 (8, 16, 24, 32, 40)

## 6. 컴포넌트 규칙

### 버튼
- Primary: navy 배경 + white 텍스트
- Secondary: white 배경 + navy 보더
- Danger: danger 배경 + white 텍스트
- disabled 상태는 명도 하향 + 클릭 비활성

### 카드
- 디자인 카드: 썸네일, 템플릿명, 추천 태그
- 주문 요약 카드: 수량, 단가, 합계, 예상 제작시간
- 장비 상태 카드: 장비명, 연결상태, 최근 응답시간

### 상태 배지
- online: success
- busy: warning
- offline: danger
- unknown: neutral

### 모달
- 결제 진행, 제작 진행, 오류 안내는 전체 포커스 모달 사용
- 모달에는 항상 닫기 또는 재시도 액션을 제공

## 7. 인터랙션 규칙

- 페이지 전환: 180~240ms 페이드 + 슬라이드
- 버튼 터치 피드백: 100ms 스케일 0.98
- 진행 상태는 단계형 스텝퍼로 표시
- 오류 발생 시 토스트와 상세 모달 동시 제공(사용자/관리자 구분)

## 8. 카피라이팅 규칙

- 사용자 화면 문구 예시
  - "원하시는 문구를 입력해 주세요"
  - "결제가 완료되면 바로 제작이 시작됩니다"
  - "장비 점검 중입니다. 잠시만 기다려 주세요"
- 관리자 화면 문구 예시
  - "프린터 응답 지연 (Timeout 5000ms)"
  - "재시도 1/3 수행 중"
  - "결제 승인 응답 수신"

## 9. 접근성/사용성

- 대비 비율 WCAG AA 이상 준수
- 모든 주요 버튼에 아이콘+텍스트 병행
- 컬러만으로 상태를 구분하지 않고 아이콘/텍스트 함께 제공
- 긴급 상황 안내는 상단 고정 배너로 노출

## 10. 데모 우선 적용 범위

1. 사용자 4단계 플로우 UI (디자인 -> 확인 -> 결제 -> 제작완료)
2. 관리자 장비 상태 대시보드
3. 작업 큐/장애 로그 테이블
4. 결제/제작 진행 모달과 오류 복구 액션
