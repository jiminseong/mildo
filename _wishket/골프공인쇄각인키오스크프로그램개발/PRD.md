# 골프공 인쇄/각인 키오스크 데모 PRD

## 1. 문서 개요

- 프로젝트명: 골프공 인쇄/각인 키오스크 데모
- 대상 폴더: _wishket/골프공인쇄각인키오스크프로그램개발
- 문서 목적: 본개발 착수 전, 화면/동작/장비연동 인터페이스를 검증 가능한 데모 범위로 정의
- 기준 기술: Next.js(Web), TypeScript, 브라우저 로컬 저장소(localStorage)

## 2. 제품 목표

- 기존 운영 키오스크의 사용자 흐름을 벤치마킹하여 핵심 동작을 재현한다.
- 하드웨어 실장비 연동 전에도, 장비 어댑터 구조와 작업 파이프라인을 데모에서 검증한다.
- 결제-출력-각인-완료까지 운영 시나리오를 실패 복구 포함하여 확인한다.

## 3. 데모 범위

### 3.1 포함 범위

- 사용자 키오스크 화면
  - 디자인 선택(문자/이미지/캐릭터)
  - 미리보기
  - 수량/옵션 선택
  - 결제 진행
  - 제작 진행 상태 표시
- 관리자 화면
  - 장비 연결 상태(프린터/결제기/커팅기/스크린)
  - 작업 큐 상태(대기/진행/완료/실패)
  - 결제 이력 조회
  - 장애 로그 조회 및 재시도
- 연동 계층
  - 장비 어댑터 인터페이스(PrinterAdapter, PaymentAdapter, CutterAdapter)
  - Mock 드라이버 + 실장비 드라이버 슬롯 구조
- 로컬 데이터
  - SQLite에 주문/결제/작업/장애 로그 저장

### 3.2 제외 범위

- 실서비스 운영 배포 자동화
- ERP/외부 정산 시스템 연동
- 다중 키오스크 중앙 관제
- 고급 분석 리포트
- Electron 패키징 및 데스크톱 배포

## 4. 사용자 유형

- 일반 사용자: 키오스크에서 디자인 선택, 결제, 제작 완료 확인
- 운영 관리자: 장비 상태 모니터링, 이력 확인, 장애 재처리

## 5. 핵심 사용자 흐름

1. 사용자: 홈 진입 -> 디자인 편집 -> 옵션 선택 -> 결제 -> 제작 상태 확인 -> 완료
2. 관리자: 관리자 로그인(데모) -> 장비 상태 확인 -> 실패 작업 재시도 -> 로그 확인

## 6. 기능 요구사항

### FR-01 디자인 편집

- 문자 입력, 기본 폰트/크기/정렬 설정 가능
- 이미지/캐릭터 템플릿 선택 가능
- 결과 미리보기 표시

### FR-02 주문/결제

- 수량, 단가, 총액 계산
- 결제 요청/응답 처리(데모 단계에서는 Mock 가능)
- 결제 성공 시 작업 큐에 제조 작업 생성

### FR-03 제조 작업 파이프라인

- 작업 상태: queued -> processing -> done / failed
- 실패 시 에러코드/메시지 기록
- 재시도 실행 시 새 작업 또는 재처리 상태 기록

### FR-04 장비 연동 추상화

- 프린터/결제기/커팅기 공통 인터페이스 제공
- 장비별 연결 확인(checkConnection), 명령 실행(execute), 상태 조회(getStatus) 표준화
- 실장비 미연결 시 Mock 어댑터로 기능 시연 가능

### FR-05 관리자 기능

- 장비 상태 대시보드
- 결제 이력 목록(시간, 금액, 결과)
- 작업 이력 목록(작업ID, 상태, 실패사유)
- 장애 로그 필터 및 재시도 버튼

### FR-06 로컬 저장

- 브라우저 로컬 저장소(localStorage) 기반 데이터 보관
  - orders
  - payments
  - jobs
  - device_logs
- 앱 재시작 후 이력 복원 가능

## 7. 비기능 요구사항

- 앱 시작 5초 이내 메인 화면 진입
- 결제/출력 요청 타임아웃 설정 가능
- 모든 장비 명령/응답 로그 timestamp 저장
- 예외 발생 시 사용자에게 안내 메시지 노출, 관리자에는 상세 로그 제공

## 8. 데이터 모델 (데모)

### Order

- amount
- status
- created_at

### Payment

- amount

### Job

- created_at

## 9. 수용 기준

1. 디자인 편집 후 주문 생성이 가능해야 한다.
2. 결제 성공 시 제조 작업이 큐에 등록되어 상태가 변경되어야 한다.
3. 실패 작업은 관리자 화면에서 재시도 가능해야 한다.
4. 앱 재시작 후 주문/결제/작업 이력이 유지되어야 한다.
5. Mock 모드에서 실장비 없이 전체 시나리오 데모가 가능해야 한다.

## 10. 마일스톤 (데모)

- M1 (1주): 화면 골격, 디자인 편집, SQLite 스키마
- M2 (2주): 결제/작업 큐, 장비 어댑터 인터페이스, Mock 드라이버
- M3 (3주): 관리자 대시보드, 로그/재시도 기능
- M4 (4주): 통합 테스트, 데모 시나리오 정리, 문서화
