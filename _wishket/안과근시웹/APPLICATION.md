# 안과 근시 데이터 관리 웹 프로그램 지원서 초안

## 기본 입력값

- 지원 금액: 700,000원
- 지원 기간: 4일
- 관련 포트폴리오: 있습니다

### 금액/기간 제안 메모

- 본 공고는 레퍼런스 기반의 핵심 기능 재현이 목표이므로, 1차 MVP 범위를 명확히 고정하면 70만원/4일 내 납품이 가능합니다.
- 기능 추가 요청(문자 알림, 통계 고도화, 다지점/다의료진 권한 체계 등)은 2차 고도화로 분리하는 방식이 일정 준수와 품질 확보에 가장 안전합니다.

## 제출용 텍스트 (복붙용)

### 지원서 본문 (지원 내용)

```text
지원 금액: 700,000원
지원 기간: 4일

- 본 프로젝트는 안과에서 사용하는 근시 데이터 기록/추적 도구로, 핵심은 “입력 정확도 + 조회 편의성 + 역할별 접근 제어”라고 이해했습니다.
- 공고의 우선순위(완성도 > 금액 > 일정)를 반영해, 1차 납품 범위를 명확히 고정한 뒤 4일 내 안정적으로 배포 가능한 구조로 제안드립니다.

<프로젝트 진행 제안>
[1차 구축 범위(MVP)]
- 사용자 진입 구조
  - 안과의사(기록 등록/관리)
  - 보호자(자녀 기록 조회)
  - 익명(둘러보기, 저장 불가)
- 환자 식별/검색
  - 이름 + 생년월일 기반 조회
  - 검색 결과 없을 시 안내 처리
- 근시 데이터 기록/조회
  - 기본 진료 기록 입력(날짜, 좌/우안 수치, 메모)
  - 환자별 시계열 이력 조회
- 대시보드/통계(기본)
  - 최근 등록 건수, 환자별 최근 기록 요약
- 기본 운영 환경
  - 서버/DB/배포 구성
  - 반응형 웹(PC/태블릿/모바일)

[2차 고도화(별도)]
- 고급 통계(기간 비교, 연령대 분포, CSV 다운로드)
- 다지점/다의료진 세부 권한
- 알림 기능(문자/이메일), 예약/상담 연동
- EMR 연동 등 외부 시스템 연계

<기술 제안>
- Frontend: Next.js + TypeScript + Tailwind CSS
- Backend/DB/Auth: Supabase (PostgreSQL, Auth, Storage)
- 배포: Vercel + Supabase 조합 또는 협의된 인프라

<예상 일정 (4일)>
- 1일차: 요구사항 확정, DB 스키마/권한 설계, 화면 골격 구축
- 2일차: 로그인/권한 분기, 환자 검색, 기록 등록 구현
- 3일차: 기록 조회/기본 대시보드, 예외 처리, 반응형 보정
- 4일차: 최종 QA, 피드백 반영, 실서버 배포 및 인수인계

<검수 및 수정 방식>
- 일 단위 데모 링크 공유로 중간 검수
- 기능 단위 체크리스트 기반 확인(로그인/검색/등록/조회/권한)
- 납품 전 최종 시나리오 테스트(의사/보호자/익명)

<유사 프로젝트 진행 경험>
- 포토 키오스크 관리용 웹 (운영자 입력 + 사용자 조회 구조의 반응형 웹 구축 경험)
- 역할 기반 접근 제어와 데이터 테이블 설계 경험
- 짧은 일정 내 MVP 우선 납품 후 고도화 단계 분리 진행 경험

<사용 기술과 툴>
- 프론트엔드: Next.js, React, TypeScript
- 스타일링: Tailwind CSS
- 백엔드/DB: Supabase, PostgreSQL
- 협업/관리: 요구사항 체크리스트, 단계별 데모 공유

<산출물>
- 배포 완료된 웹사이트
- 프론트/백엔드 소스 코드 원본
- DB 스키마 및 환경 변수 가이드
- 운영자 기준 기본 사용 가이드
```

### 유사 프로젝트 진행 경험
<개요>
- 프로젝트명: 포토 키오스크 관리용 웹
- 수행 역할: 기획, 디자인, 화면 설계, 풀스택(프론트엔드,백엔드) 구현, 운영자 기능 연동
- 구현 범위: 운영자 데이터 입력 화면, 사용자 조회 화면, 반응형 UI, 배포 환경 구성

<유사 포인트>
- 운영자 입력과 사용자 조회를 분리한 구조로 권한별 접근 흐름을 명확히 설계
- 테이블 중심 데이터 관리(등록/수정/조회)와 검색 조건 처리 경험 보유
- 단기 MVP 납품 후 고도화 항목을 분리해 일정 지연 없이 인수인계한 경험 보유

<이번 프로젝트 적용>
- 의사/보호자/익명 사용자 분기, 환자 검색, 기록 등록/조회 구조를 동일한 방식으로 빠르게 재현 가능


## 작성 메모

- 공고 정보(예산 70만원, 기간 4일, 레퍼런스 유사 구현)를 기준으로 1차 MVP 범위를 구체화했습니다.
- 레퍼런스에서 확인 가능한 핵심 플로우(의사/보호자/익명, 환자 검색, 기록 조회)를 제안 본문에 반영했습니다.
- 일정 리스크를 줄이기 위해 2차 고도화 항목을 분리해 완성도와 납기 가능성을 동시에 확보하도록 구성했습니다.
