# 선거 후보자 프로필 및 공약 홍보 웹페이지 DESIGN SYSTEM

## 1. 문서 개요

- 프로젝트명: 선거 후보자 프로필 및 공약 홍보 웹페이지 구축
- 작성일: 2026-03-15
- 적용 방식: 편집형 레이아웃, 큰 타이포, 넓은 여백 원칙만 참고하고 컬러/토큰/브랜드 표현은 본 프로젝트 전용으로 설계한다.
- 목표: 선거 홍보용으로 **중립적이고 신뢰감 있는 적청 혼합 표현**을 갖춘 독립 디자인 시스템을 정의한다.

## 2. 디자인 방향

### 2.1 핵심 원칙

1. **Editorial Trust**
   - 가운데 정렬 남발 대신 좌측 정렬과 넉넉한 여백으로 신뢰감을 만든다.
2. **Neutral Civic Balance**
   - 정당이 확정되지 않은 상태를 전제로, 빨강과 파랑을 모두 쓰되 어느 한쪽이 사이트 전체를 지배하지 않게 한다.
3. **Profile First**
   - 선거 홍보 페이지는 장식보다 후보자 얼굴, 이름, 이력, 공약이 먼저 읽혀야 한다.
4. **Mobile First**
   - 모바일에서 읽기 어렵다면 데스크톱 완성도도 실패로 본다.

### 2.2 무드 키워드

- 차분한 신뢰
- 시민적 긴장감
- 사람 중심 사진
- 공보물보다 읽기 쉬운 웹 편집

## 3. 컬러 시스템

### 3.1 Base Palette

본 프로젝트 전용 중성 팔레트다. 배경은 순백을 기본으로 하고, 구획 구분은 아주 옅은 회색과 얇은 라인으로 처리한다.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `--color-base` | `#FFFFFF` | 전체 배경 |
| `--color-surface` | `#F7F8FA` | 카드/구획 배경 |
| `--color-text-primary` | `#151922` | 기본 본문/헤드라인 |
| `--color-text-secondary` | `#5B6472` | 보조 설명 |
| `--color-border` | `#E2E6EC` | 구분선, 테이블 라인 |

### 3.2 Election Accent Palette

정당 고정색이 아니라, 선거 문맥을 위한 **시민적 적청 포인트**로 사용한다.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `--color-civic-blue` | `#42648F` | 신뢰, 데이터, 구조적 강조 |
| `--color-civic-blue-soft` | `#DCE6F4` | 블루 배지, 연한 배경 |
| `--color-civic-red` | `#C15E52` | 열정, 참여, 핵심 포인트 |
| `--color-civic-red-soft` | `#F0DDD8` | 레드 배지, 연한 배경 |
| `--color-civic-navy` | `#24364B` | 헤더, 강한 CTA, 깊이감 |

### 3.3 사용 비율

- 중성 배경/텍스트/라인: 70%
- 블루 계열 포인트: 15%
- 레드 계열 포인트: 15%

### 3.4 사용 규칙

- 한 화면에 빨강과 파랑을 모두 쓸 수 있지만, **동일 컴포넌트 안에서는 주색 1개만 선택**한다.
- 파랑은 구조와 신뢰 표현에 사용한다.
- 빨강은 대표 공약, 참여 유도, 중요 배지에 사용한다.
- 전체 배경은 흰색을 유지하고, 포인트 컬러는 CTA, 라벨, 구분선 강조에만 제한한다.
- 선거 사이트라도 순수 원색 `#FF0000`, `#0000FF` 계열은 금지한다.
- 배경 전체를 파랑이나 빨강으로 채우지 않는다.

## 4. 타이포그래피

### 4.1 폰트

- 메인: `Pretendard Variable`
- 폴백: `Pretendard, system-ui, sans-serif`
- 다른 제목용 서체는 사용하지 않는다.

### 4.2 타입 스케일

| 역할 | 모바일 | 데스크톱 | 비고 |
| --- | --- | --- | --- |
| Hero Title | `40/44` | `72/78` | 후보자명 또는 대표 카피 |
| Hero Meta | `15/22` | `18/28` | 출마 정보, 지역, 슬로건 |
| Section Title | `28/34` | `40/48` | 섹션 헤더 |
| Card Title | `20/28` | `24/32` | 공약/프로필 카드 제목 |
| Body Large | `17/28` | `18/30` | 긴 소개문 |
| Body | `15/24` | `16/28` | 일반 본문 |
| Label | `12/18` | `13/20` | 태그, 메타 정보 |

### 4.3 문장 규칙

- `word-break: keep-all` 기준
- 한 문단은 모바일 기준 3줄 안팎으로 쪼갠다.
- 긴 공약 설명은 4개 이하 불릿 또는 2단락 이내로 정리한다.
- 굵기 변화만으로 위계를 만들고, 과한 자간 장식은 쓰지 않는다.

## 5. 레이아웃 시스템

### 5.1 컨테이너

- 최대 너비: `1320px`
- 기본 좌우 패딩:
  - 모바일: `20px`
  - 태블릿: `28px`
  - 데스크톱: `40px`

### 5.2 그리드

- 모바일: 4컬럼
- 태블릿: 8컬럼
- 데스크톱: 12컬럼

### 5.3 섹션 간격

- 모바일 섹션 간격: `72px`
- 태블릿 이상: `96px`
- 히어로 하단은 최소 `88px`, 데스크톱에서 `120px` 확보

### 5.4 정렬 원칙

- 텍스트는 좌측 정렬이 기본
- 히어로는 좌측 텍스트 + 우측 인물 사진 또는 카드형 보조 정보
- 섹션 내부 카드들은 가운데 정렬보다 기준선이 맞는 정렬을 우선

## 6. 반응형 규칙

### 6.1 모바일 `360px~767px`

- 후보자 이름, 슬로건, CTA를 첫 화면 안에 최대한 수납
- 프로필, 공약, 갤러리는 모두 1열
- 상단 메뉴는 축약하고 섹션 점프는 드로어 또는 가로 스크롤 탭으로 처리
- 하단 고정 CTA는 필요할 때만 사용

### 6.2 태블릿 `768px~1023px`

- 히어로 2단 분할 가능
- 공약 카드는 2열
- 프로필 요약 카드와 대표 사진을 분리 배치

### 6.3 데스크톱 `1024px+`

- 비대칭 편집 레이아웃 적극 사용
- 히어로는 타이포 중심, 이미지/프로필 정보는 보조 축으로 배치
- 공약 영역은 요약 카드 + 상세 패널 구조 권장

## 7. 컴포넌트 규칙

### 7.1 Header

- 얇은 보더 기반 고정 헤더
- 배경은 `base` 또는 반투명 화이트 계열
- 메뉴는 4~6개 이내
- CTA 버튼은 `civic-navy` 또는 `civic-red` 중 하나만 선택

### 7.2 Hero

- 후보자 얼굴이 선명한 실사 사용
- 타이포 영역과 사진 영역 비율은 `5:7` 또는 `6:6`
- 상단 라벨은 `civic-blue-soft` 또는 `civic-red-soft`
- 메인 CTA와 서브 CTA는 서로 다른 포인트 색을 사용해 적청 균형을 만든다.

### 7.3 Profile Card

- 카드 배경은 `surface` 또는 `white`
- 반경은 `12px`
- 얇은 라인과 메타 정보 중심 구성
- 출생연도, 학력, 경력, 소속 등은 2열 정보표 또는 스택 카드로 정리

### 7.4 Pledge Card

- 핵심 공약 카드는 3~5개가 가장 적절
- 카드 상단에 카테고리 라벨 배치
- 블루는 정책/행정/신뢰 톤, 레드는 민생/참여/변화 톤에 매칭
- 호버 시 그림자보다 보더색 변화와 미세 이동을 사용

### 7.5 Pledge Detail

- 아코디언 또는 확장 패널 사용
- 각 항목은 아래 구조를 유지
  - 무엇을 하려는가
  - 왜 필요한가
  - 누구에게 도움이 되는가
  - 기대 효과

### 7.6 Gallery

- 4:5, 3:4 비율 이미지를 우선 사용
- 너무 많은 썸네일을 한 번에 보여주지 않는다.
- 캡션은 장소/맥락 중심으로 짧게

### 7.7 Buttons

- 기본 반경: `999px` 또는 `12px`
- 최소 높이: `44px`
- Primary: `civic-navy` 또는 `civic-blue`
- Secondary: 흰색 배경 + 보더 + `civic-red` 또는 `civic-blue` 텍스트
- Hover: `translateY(-1px)` 수준의 미세 이동

## 8. 이미지 및 아트 디렉션

- 스톡 이미지 사용 금지
- 실제 활동 사진, 인물 사진, 현장 컷 사용
- 이미지 보정은 과포화 금지, 피부톤 자연스럽게 유지
- 얼굴 클로즈업만 반복하지 말고, 군중/현장/악수/설명 장면을 함께 섞는다.
- 배경 제거 인물컷보다 현장감 있는 원본 사진이 우선이다.

## 9. 모션 원칙

- 섹션 등장 애니메이션은 `200ms~400ms` 범위의 짧은 fade/slide만 허용
- 스크롤 패럴랙스, 과한 블러, 대형 비디오 배경 금지
- 인터랙션 목적은 화려함이 아니라 정보 인지 보조여야 한다.

## 10. 접근성 원칙

- 텍스트와 배경은 WCAG 대비를 충족해야 한다.
- 색만으로 상태를 구분하지 않는다.
- 포커스 링을 명확히 노출한다.
- 사진 캡션과 ALT는 서로 다른 역할로 작성한다.
- CTA는 행동이 분명한 문구를 사용한다.

## 13. 금지 사항

1. 빨강 또는 파랑 한 색으로 사이트 전체를 칠하는 방식
2. 네온, 글로우, 과한 그라데이션
3. 모든 텍스트를 중앙 정렬하는 템플릿형 랜딩
4. 스톡 사진, AI 느낌이 강한 합성 비주얼
5. 공약 내용을 긴 문단으로만 쌓는 방식
6. 과도한 원형 버튼, 과도한 그림자, 유리질감 카드

## 14. 최종 원칙

- 이 프로젝트는 공보물 느낌보다 **신뢰되는 편집형 웹페이지**에 가까워야 한다.
- 빨강과 파랑은 각각 참여와 신뢰를 상징하는 보조 언어로 사용하되, 어느 한쪽도 사이트 전체를 지배하지 않게 운용한다.
- 폰트는 반드시 Pretendard 기반으로 통일하고, 반응형에서 정보 위계가 무너지지 않도록 설계한다.
