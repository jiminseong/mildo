# 기업 홈페이지 DESIGN SYSTEM

## 1. 디자인 방향

- 키워드: Modern, Clean, Premium, Trust
- 목표: 정보 전달력과 브랜드 신뢰를 동시에 확보
- 원칙: 과한 장식보다 구조적 명확성, 제품 중심 가독성, 일관된 위계

## 2. 톤앤매너

- 시각 톤: 밝고 정돈된 배경 + 딥 계열 포인트 컬러
- 카피 톤: 짧고 명확한 문장, 과장 표현 최소화
- 인터랙션 톤: 적극적인 모션 활용으로 프리미엄 인상 강화, 다만 정보 가독성을 해치지 않도록 제어

## 5. 레이아웃 규칙

- 콘텐츠 최대 폭: 1200px
- 기본 그리드: 12 columns
- 브레이크포인트:
  - Mobile: 360~767
  - Tablet: 768~1023
  - Desktop: 1024+

### 섹션 간격

- Desktop: 88px
- Tablet: 72px
- Mobile: 56px

## 6. 컴포넌트 가이드

### 버튼

- Primary: 진한 배경 + 흰색 텍스트
- Secondary: 흰 배경 + 라인 보더
- Tertiary: 텍스트 링크형
- 라벨은 동작 중심 문구 사용(예: "제품 문의하기", "상세보기")

### 카드

- 제품 카드: 썸네일, 카테고리, 제품명, 요약, CTA
- 공지 카드: 제목, 요약, 등록일
- 관리자 리스트 카드: 상태 배지 + 핵심 필드 요약

### 폼

- 모든 입력 필드에 label 표시
- 에러 메시지는 필드 하단에 즉시 노출
- 필수 항목은 `*` 표기 + 안내 문구 제공

## 7. 모션 가이드

- 핵심 원칙: 모든 주요 섹션은 의미 있는 진입 모션을 가진다.
- 기본 전환: 220~420ms
- 강조 전환: 500~700ms (히어로, 핵심 CTA)
- 이징: ease-out, cubic-bezier(0.22, 1, 0.36, 1) 병행

### 페이지별 필수 모션

- 메인 히어로: 배경 -> 타이틀 -> 서브카피 -> CTA 순차 등장(stagger)
- 섹션 진입: 스크롤 트리거 reveal(translateY + opacity)
- 제품/뉴스 카드: hover 시 scale, shadow, accent line 확장
- 상세 페이지: 이미지와 본문 블록의 교차 진입 모션
- 문의 폼: 단계별 포커스 강조 및 제출 성공 모션
- 관리자 화면: 저장/수정/삭제 시 상태 배지 트랜지션

### 구현 가이드

- transform/opacity 중심 애니메이션 우선
- 레이아웃 이동이 큰 속성(top/left/width/height) 애니메이션은 지양
- 동일 페이지 내 모션 딜레이 간격은 60~120ms 유지
- 긴 리스트는 최초 8~12개 요소까지만 stagger 적용

### 접근성

- prefers-reduced-motion 활성화 시 모션 강도를 축소하거나 제거
- 필수 정보는 모션 없이도 항상 즉시 인지 가능해야 함

## 8. 이미지/아이콘 가이드

- 이미지 스타일: 제품 실사 중심, 과도한 필터 지양
- 비율: 제품 썸네일은 4:3 기준
- 아이콘: 단순 선형 아이콘 일관 사용

## 9. 접근성 가이드

- 텍스트 대비 WCAG AA 이상
- 키보드 포커스 링 명확히 표시
- 링크/버튼의 시각적 구분 명확화
- 폼 요소 ARIA 속성 및 오류 상태 전달

## 10. 페이지별 우선순위

1. 메인: 기업 메시지 + 제품 하이라이트 + 문의 CTA
2. 소개: 신뢰 정보(비전/연혁/위치)
3. 제품: 탐색 효율(카테고리/검색/상세)
4. 문의: 최소 입력으로 전환
5. 관리자: 빠른 등록/수정/상태 확인
