# 심리검사 서비스 프로토타입 PRD (최종)

## 1. 문서 개요

- 프로젝트명: 연구과제용 웹 심리검사 프로토타입
- 개발 기간: 30일
- 지원 디바이스: PC Web 우선
- 목표: 아날로그/엑셀 기반 검사 절차를 웹으로 전환해 접근성과 계산 자동화 가능성을 검증

## 2. 배경 및 목표

- 현재 아날로그/엑셀 기반 운영으로 참여/분석 효율이 낮음
- 무회원 접근 + 즉시 결과 제공 + 관리자 로그 확인이 가능한 프로토타입 필요
- 상용화 전 단계에서 UI보다 계산 로직 재현성과 리포트 신뢰도 검증이 핵심

## 3. 과업 범위

### 3.1 포함 범위 (MVP)

- 사용자 검사 기능
  - 무회원 검사 진입
  - 순차 문항 응답
  - 결과 확인
  - 결과 PDF 다운로드
- 결과 분석/리포트 생성
  - 응답 저장 및 가공
  - 엑셀 계산식 자동화
  - t-test 처리 로직 반영(합의 범위)
  - 결과 리포트 생성
- 관리자 기능
  - 접속 로그 조회 페이지
- 서버/DB/인프라 구성

### 3.2 제외 범위 (후속)

- 회원 시스템 및 고도화 권한
- 고급 통계 대시보드
- 대규모 콘텐츠 CMS
- 다국어 및 마케팅 확장

## 4. 상세 기능 요구사항

### 4.1 사용자 검사 기능

1. 검사 시작
- 누구나 링크로 접근 가능
- 기본 안내 후 즉시 시작

2. 검사 진행
- 문항 순차 응답 인터페이스
- 진행률 표시
- 필수값 검증

3. 결과 확인
- 완료 직후 결과 리포트 제공
- 척도별 점수/해석 문구 표시

4. 결과 다운로드
- PDF 파일 저장 제공

### 4.2 분석 및 리포트

- 응답 데이터 저장
- 엑셀 계산 공식 이관
- 통계 처리(t-test) 자동화
- 리포트 페이지 생성

### 4.3 관리자 기능

- 접속 로그 확인
- 완료 여부/상태 확인

## 5. 정보 구조 (IA)

- `/` 검사 안내/시작
- `/assessment` 검사 진행
- `/report/[id]` 결과 리포트
- `/admin/logs` 관리자 로그

## 6. 데이터 모델 (초안)

### Question
- id
- order
- text
- scale
- reverseScored

### ResponseSession
- id
- startedAt
- completedAt
- status
- answers[]

### ScoreResult
- sessionId
- rawScores
- tScores
- interpretation
- generatedAt

### AccessLog
- id
- sessionId
- eventType
- path
- createdAt

## 7. 비기능 요구사항

- 정확성: 엑셀 샘플 결과와 웹 결과 일치
- 성능: 주요 페이지 로딩/응답 안정성 확보
- 사용성: PC 환경에서 명확한 검사 흐름
- 유지보수: 계산 모듈 분리 및 테스트 가능한 구조

## 8. 수용 기준 (Acceptance Criteria)

1. 무회원 검사 진입 가능
2. 응답 완료 후 결과 리포트 즉시 표시
3. PDF 다운로드 동작
4. 엑셀 기준 샘플 검증 통과
5. 관리자 로그 페이지 조회 가능
6. `npm run lint`, `npm run build` 통과

## 9. 일정 계획

- 1주차: 요구사항/엑셀 분석, 데이터 모델 확정
- 2주차: 검사 흐름 및 리포트 UI 구현
- 3주차: 계산 모듈 + PDF + 관리자 로그 구현
- 4주차: QA, 검증, 배포, 산출물 정리

## 10. 리스크 및 대응

- 엑셀 수식 복잡도: 우선순위 수식부터 단계 이관
- 통계 기준 차이: 샘플 비교표로 상호 검증
- 해석 문구 미확정: 임시 구간 문구 후 최종 교체

## 11. 최종 산출물

- 전체 소스코드
- 서버/DB 설계 내역서
- 기능 명세/운영 가이드
- 엑셀-웹 계산 검증 결과 문서
