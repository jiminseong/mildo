# 심리검사 로컬 데모 전용 실행 계획

기준 문서: OVERVIEW.md, PRD.md, TASK.md
적용 범위: psych-assessment-prototype
목표: 클라이언트 요청 핵심 기획을 로컬에서 전부 시연 가능 상태로 구현

## 1) 데모 구현 원칙

- 로컬 단일 실행으로 전체 흐름 시연 가능
- 계산 로직은 임의 모델로 구성하되, 구조는 실제 이관 가능한 형태로 분리
- 서버/DB는 로컬 데모 단계에서 mock 데이터 레이어로 대체
- 기능별 완료 여부를 체크리스트로 관리

## 2) 로컬 데모 범위(전체)

- 무회원 검사 진입
- 순차 문항 응답 UI
- 진행률 및 필수값 검증
- 결과 리포트(척도별 점수, 종합 지수, 해석)
- PDF 저장(브라우저 인쇄)
- 엑셀 이관 대체용 임의 계산 로직
- 관리자 로그(조회/상태 필터/정렬)
- 로컬 실행 및 lint/build 검증

## 3) 계산 로직(임의 모델)

- 3개 척도: 스트레스, 회복탄력성, 정서안정성
- 엑셀 유사 외부 설정 테이블 분리:
  - 척도 테이블(SCALE_TABLE): 기대값/가중치/컷오프
  - 문항 테이블(QUESTION_TABLE): 문항 ID/문항 텍스트/순서
  - 문항-척도 매핑 테이블(QUESTION_SCALE_MAP): 역채점/문항 가중치
  - 종합 위험지수 컷오프(RISK_INDEX_CUTOFFS)
- 문항별 척도 매핑 + 역채점 반영
- 척도별 원점수 평균(1~5), 100점 환산 점수 계산
- 종합 위험지수 계산:
  - 위험지수 = 스트레스100 * 0.5 + (100 - 회복탄력성100) * 0.25 + (100 - 정서안정성100) * 0.25
- 위험구간 해석:
  - 70 이상: 고위험
  - 40 이상: 주의
  - 40 미만: 안정
- 비교표 제공:
  - 각 척도의 기대기준(임의 기준값) 대비 차이

## 5) 수동 점검 시나리오

1. 홈에서 검사 시작
2. 문항 선택 없이 완료 시도 -> 차단 확인
3. 모든 문항 응답 후 결과 리포트 이동
4. 척도별 점수/위험지수/해석/비교표 확인
5. PDF 저장 클릭
6. 관리자 로그에서 상태 필터(완료/진행중) 및 정렬 변경 확인

## 6) 실행 명령

- npm run dev
- npm run lint
- npm run build
