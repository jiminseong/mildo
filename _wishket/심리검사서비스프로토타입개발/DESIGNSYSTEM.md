# 심리검사 서비스 프로토타입 디자인 시스템 (최종)

## 1. 디자인 원칙

1. 신뢰성: 연구 도구 특성에 맞는 차분한 시각 톤
2. 명확성: 문항-응답-결과 흐름이 즉시 이해되는 구조
3. 일관성: 검사/리포트/관리자 화면 컴포넌트 규칙 통일
4. 가독성: 해석 문구 중심 콘텐츠의 읽기 피로 최소화

## 2. 시각 방향

- 대상: PC Web 우선
- 성격: 연구/실험용 서비스 인터페이스
- 지양: 과도한 장식, 과한 애니메이션, 마케팅 랜딩형 과포장

## 3. 컬러 토큰

- `--bg`: #f7f8fb
- `--surface`: #ffffff
- `--text-primary`: #1f2937
- `--text-secondary`: #4b5563
- `--line`: #d1d5db
- `--primary`: #1d4ed8
- `--primary-soft`: #dbeafe
- `--success`: #15803d
- `--warning`: #b45309
- `--danger`: #b91c1c

## 4. 타이포그래피

- 폰트: Pretendard, system sans-serif
- H1: 32/40, 700
- H2: 24/32, 700
- H3: 20/28, 600
- Body: 16/26, 400
- Small: 14/22, 400
- Caption: 12/18, 400

## 5. 레이아웃 규칙

- 최대 폭: 1200px
- 기본 좌우 여백: 24px
- 섹션 간격: 32~48px
- 카드 패딩: 20~24px

## 6. 컴포넌트 규칙

### 6.1 Button
- Primary: 검사 시작/제출/다운로드
- Secondary: 이전/보조 액션
- Danger: 초기화/삭제

### 6.2 Form
- 입력 라벨 필수
- 필수값 검증 메시지 즉시 노출
- 키보드 포커스 이동 가능

### 6.3 Card
- 문항 카드
- 리포트 요약 카드
- 로그 카드
- 동일 보더/패딩 규칙 유지

### 6.4 Status Badge
- 상태값: 진행중/완료/오류/검토필요
- 색상 + 텍스트 병기

## 7. 화면별 UX 가이드

### 검사 화면
- 상단: 진행률 + 현재 문항 번호
- 중앙: 문항 텍스트 + 선택지
- 하단: 이전/다음/제출 버튼

### 결과 리포트
- 상단: 세션 정보
- 중단: 척도별 점수 및 해석
- 하단: PDF 다운로드

### 관리자 로그
- 테이블 중심 정보 배치
- 필터: 기간/상태
- 핵심 컬럼: 세션ID, 접속시각, 상태, 완료여부

## 8. 접근성 기준

- 텍스트 대비 4.5:1 이상
- 모든 입력 필드 라벨 연결
- 오류 메시지 텍스트 제공
- 키보드 전용 조작 가능

## 9. 인터랙션 기준

- 전환 속도: 150~220ms
- 로딩/계산 중 상태 표시
- 완료 액션 후 명확한 피드백 제공

## 10. 적용 범위

- 와이어프레임
- UI 구현 기준
- QA 점검표
- 후속 고도화 시 공통 UI 기준점
